# `docs/prd/` - 제품 요구사항

> 프로젝트가 해결할 문제, 사용자, 범위, 성공 기준과 비범위를 정의하는 제품 기준 문서입니다.

현재 기준은 [`PRD_v0.6.md`](PRD_v0.6.md)입니다.

## 사용 원칙

- 기능 구현 전에 사용자 가치와 범위를 확인합니다.
- 기술 상세는 `functional-spec/`와 `architecture/`로 분리합니다.
- 버전을 올릴 때 기존 PRD를 덮어쓰는 대신 변경 이력을 명확히 남깁니다.
- 구현 여부는 PRD가 아니라 `functional-spec/IMPLEMENTATION_STATUS.md`를 함께 봅니다.
