## 1. Problem Statement (문제 정의)

### 1.1 Current State (현재 상황 — 출처 명시 통계)

한국에는 **2024년 12월 기준 약 1,410만 명** 의 개인 주식투자자가 있다 (한국예탁결제원, 출처 ①). 대한민국 성인 3명 중 1명(32%)이 주식 시장에 참여하고 있고, 토스증권·카카오페이증권 같은 모바일 증권 앱의 보급으로 진입 장벽은 크게 낮아졌다 — 토스증권 가입자만 **2025년 2월 기준 860만 명**으로 1년 만에 30% 이상 증가했다 (출처 ③).

그러나 *결정의 근거를 마련하는* 영역에서는 정보 비대칭이 크다. 증권사의 PB 리포트(7~50쪽 종합 분석 보고서)는 기관·고액 자산가에게 우선 제공되고, 일반 개인투자자는 유튜브·종목 토론·단편 뉴스로 의사결정한다.

그 결과는 정량 데이터로도 확인된다:
- **2024년 국내 개인투자자 358만 9,689명 분석 결과 평균 수익률 −16.77%** (출처 ②). "1억 원을 넣었으면 1,677만 원을 잃은 셈" 이라는 머니투데이 표현이 상황을 압축한다.
- 동일 기간 해외 주식 투자 평균 수익률 +32.08%
- 학술 연구(자본시장연구원 보고서)에서도 **개인투자자는 시장 수익률을 지속 하회** 하는 경향이 일관되게 관찰됨

### 1.2 Root Cause (근본 원인 가설)

이탈/실패 사용자 인터뷰는 우리에게 직접 없으나, 자본시장연구원 보고서·토스증권 사용자 발표·머니투데이 분석을 교차 정리하면 개인투자자가 *분석 단계에서 좌절하는* 원인은 셋으로 수렴한다:

| 가설 | 근거 | 영향도 | 비고 |
|------|------|--------|------|
| **A. 근거 없는 결정** — 매수/매도를 직관·소문으로 결정 | 2024년 개인투자자 평균 수익률 −16.77% (출처 ②) + 자본시장연구원 다수 연구에서 시장 하회 | 높음 | **이번 PRD에서 검증 (Phase 1+2)** |
| B. 정보 과부하 — 뉴스·공시·차트가 너무 많아 우선순위 못 정함 | 토스증권 사용자 조사: 2030 비중 68% — 모바일에서 짧은 시간에 결정 | 중간 | A 가설 안에 포함 (RAG로 함께 해결) |
| C. 포트폴리오 컨텍스트 무시 — 일반 추천이 내 보유와 안 맞음 | 학술 연구: 개인 거래 회전율 연 270%+ → 분산·집중도 고려 부족 | 중간 | A 가설과 함께 해결 (Strategist Agent로 매칭) |

**가설 A 선택 근거:**
- 영향도가 가장 크고, 부트캠프 5주 학습(RAG·ReAct·평가 하네스·금융 가이드라인)을 직접 동원할 수 있는 영역
- B/C는 A 해결의 부산물로 같이 풀 수 있음 (Qual Worker가 뉴스 우선순위, Strategist가 포트 매칭 담당)
- "단순 LLM 데모"가 아닌 **실제 의사결정에 쓸 수 있는 시스템** 을 만드는 것이 부트캠프 결과물의 차별화 핵심

### 1.3 Insight (PM 결론)

> **개인투자자가 진짜 필요로 하는 것은 "BUY/HOLD/SELL 한 줄 정답" 이 아니라, *내 포트폴리오 컨텍스트* 에서 *왜 그런 결정인지* 5개년 근거와 함께 설명해주는 PB 수준의 보조 시스템이다.**

부트캠프 5주 학습(W1 RAG·W2 평가·W3 ReAct멀티에이전트·W4 Streamlit·W5 금융AI 가이드라인) 전부를 이 시스템에 동원할 때 학습 가치와 시장 가치가 정렬된다.

## 3. Target User (대상 사용자)

### 3.1 Primary User

**한국 KOSPI/KOSDAQ에 직접 투자하는 개인투자자** 중, 다음 조건을 동시에 만족하는 사람:
- 투자 경험 1~10년 (완전 초보 ❌, 전업 트레이더 ❌)
- 보유 종목 3~15개 (분산은 하지만 전문가급은 아님)
- 단일 종목 매수/매도/홀드 결정에 *최소한의 근거* 를 원함

### 3.2 Context (이 사용자가 언제·어디서 우리 시스템을 만나는가)

- 점심·퇴근 후 모바일/PC로 증권 앱을 보다가 *결정을 못 내리는 순간*
- 유튜브·뉴스·종목 토론에서 본 정보를 *내 포트와 어떻게 연결할지 고민하는 순간*
- 보유 종목의 실적 발표·중대공시·매크로 이벤트(FOMC 등) 직전·직후

### 3.3 현재 행동 (지금은 어떻게 해결하고 있는가)

- 유튜브·블로그·종목 토론 5~10개를 흩어서 읽는다
- 증권사 앱 차트·재무 탭을 단편적으로 본다
- 직관/감/소문으로 결정한다 → 평균 수익률 −16.77% (출처 ②)

### 3.4 데모 페르소나 — 우선순위 확정 (사용자 결정)

> **시연 우선순위: 1순위 박민호(균형형) → 2순위 김재현(적극형) → 3순위 이지영(안정형)**
> **MVP 섹터 한정: 반도체 · 금융 · 소비재 (다른 섹터는 v2 후속)**

|------|----------|-------------------------------|-----------------|-----------------|

| | 목표 수익률 | 12~15% | 18%+ | 7~10% |

| **고급 기능 (6개)** | 5개년 밸류에이션 / 산업·정성 분석 / 동종업계 횡비교 / BUY·HOLD·SELL 권유 / 종목 추천 (Curator) / PB 리포트 다운로드 | 부트캠프 학습 5주차(RAG·ReAct멀티에이전트·평가)를 직접 동원하는 차별화 영역. Phase 2 (11주차) 완료 |
| **에이전트 6종** | Curator / Qual Worker ★ / Quant Worker / Competitor / Strategist & Synthesizer / Guardrail & Evaluator | 위 11개 기능을 책임지는 단위. 강의 W1~W5 직접 매핑 |
| **횡단 가드레일** | PII 마스킹 / 욕설·광고 정규화 / 책임 고지 자동 부착 | W5 금융보안원 가이드라인 준수 (강사 피드백 #1·#4) |

### 4.2 Out of Scope (이번 PRD에서 *하지 않음*)

> 강사 가이드: "AI가 In에 넣으라는 걸 다 넣으면 위시리스트"

| 기능 | 이유 |
|------|------|
| **What Changed 델타 리포트** (직전 분석 대비 자동 diff) | 12주 일정에서 가치 대비 구현 부담 큼. v2 (Phase 3) 후속 |

| **글로벌 Peer 비교** (Yahoo Finance) | 데이터팀 MVP 카테고리에 글로벌 Peer 미포함. 국내 동종 ≥3개 비교만 |

| **모바일 앱** | Streamlit Cloud 웹만. 모바일은 v2 |
| **다국어 지원** | 한국 시장·한국어 사용자 대상. 영어는 v2 |
| **소셜·커뮤니티 기능** | 핵심 가치(개인 의사결정 지원)와 무관 |
| **반도체·금융·소비재 외 섹터** | 단기간 광범위 구현 시 품질 확보 어려움. v2 후속 |

## 5. User Stories

> 강사 가이드: 1 Story = 1 행동, *so that(이유)* 가 핵심

### 5.1 기본 기능 영역

1. **As a** 첫 방문 신규 유저, **I want to** 이메일·비밀번호로 가입하고 투자성향(보수/균형/적극)과 관심 섹터(반도체/금융/소비재)를 등록하고 싶다, **so that** 매번 입력하지 않아도 내 성향에 맞춘 추천을 받을 수 있다.

2. **As a** 가입한 유저, **I want to** 내 보유 종목·매수가·수량을 한 번에 입력(또는 CSV 업로드)하고 싶다, **so that** 나중에 분석할 때마다 다시 입력하지 않아도 된다.

3. **As a** 종목명을 정확히 모르는 유저, **I want to** "삼성전자" 또는 "005930" 으로 검색해서 자동완성으로 종목을 찾고 싶다, **so that** 티커를 외우지 않아도 빠르게 분석을 시작할 수 있다.

4. **As a** 종목을 처음 보는 유저, **I want to** 해당 종목의 1년 차트·PER/PBR·최근 7일 뉴스 헤드라인을 한 화면에서 보고 싶다, **so that** 본격 분석 전에 종목이 어떤 회사인지 빠르게 파악할 수 있다.

5. **As a** 여러 종목을 보유한 유저, **I want to** 내 포트폴리오 전체의 평가손익·비중·섹터 분포를 한눈에 보고 싶다, **so that** 어디가 무거운지(집중도) 어디가 가벼운지 알 수 있다.

### 5.2 고급 기능 영역

6. **As a** 종목 매수/매도를 고민하는 유저, **I want to** 그 종목의 보수/기본/낙관 3 시나리오로 5년 적정주가를 보고 싶다, **so that** 단기 변동에 흔들리지 않고 *내가 얼마에 사고 팔지* 기준선을 세울 수 있다.

7. **As a** 뉴스를 다 못 챙긴 유저, **I want to** 그 종목의 최근 호재·악재 뉴스가 출처와 함께 정리된 분석 페이지를 보고 싶다, **so that** 직접 뉴스 10개를 안 읽어도 핵심을 알 수 있고, 의심스러우면 출처 클릭으로 원문을 확인할 수 있다.

8. **As a** 비교를 좋아하는 유저, **I want to** 그 종목과 같은 산업의 국내 ≥3개 경쟁사 PER·ROE·매출성장을 표로 보고 싶다, **so that** 절대 수치가 아닌 *동종업계 대비 위치* 로 판단할 수 있다.

9. **As a** 결정을 내려야 하는 유저, **I want to** "BUY 78% / 적합도 ★★★★ / 현금 30% 중 5%p 매수 권장" 같은 한 줄 결론을 보고 싶다, **so that** 짧은 시간 안에 결정할 수 있고, 의심스러우면 *왜 그런 결론인지* 4가지 근거(WHAT·HOW MUCH·WHY·RISK)를 펼쳐서 볼 수 있다.

10. **As a** 종목을 정하지 못한 유저, **I want to** 시장 스크리닝과 내 성향에 맞춘 5~10개 후보 종목(반도체·금융·소비재 한정)을 추천 사유와 함께 받고 싶다, **so that** "오늘 뭐 사지?" 막연한 상태에서 분석 시작점을 잡을 수 있다.

11. **As a** 분석 결과를 동료/가족과 공유하려는 유저, **I want to** 7페이지 PB 양식 리포트를 PDF·DOCX로 다운로드하고 싶다, **so that** 공유받은 사람도 *근거 전체* 를 확인할 수 있고, 필요 시 본문을 직접 수정해서 활용할 수 있다.

**범위:** 기본 기능 5개 완전 동작 + 인프라 (Postgres + pgvector + LangSmith + 데이터팀 6 카테고리)

## 7. Edge Cases (예외 상황 처리)

### E1. 데이터 부재
- **상장 폐지·신규 상장으로 5y 재무 미확보** → "데이터 부족 — 분석 불가" 안내. 가용 연도 + 부족 연도 명시
- **DART 사업보고서 미제출 종목** → 정량 분석만. 정성 분석은 "공시 데이터 없음" 표기

### E2. 시장 이벤트
- **결과 출력 직전 24시간 내 가격 ±10% 급변동** → Trustworthiness Guard가 결과 보류 + "시장 변동성 큼, 재분석 권장" 안내
- **장 마감 후 분석 요청** → 마감 후 종가 기준 분석. 다음 거래일 시초가 예측 명시 제외

### E3. 사용자 입력
- **포트폴리오 비중 합계 ≠ 100%** → 자동 정규화 + 경고 표시 ("비중 합이 95%입니다. 5% 누락 처리됩니다")
- **존재하지 않는 종목 코드 입력** → "종목 없음 — 자동완성 제안" 표시
- **MVP 섹터 외 종목 입력 (예: 셀트리온/현대차)** → "MVP 섹터(반도체/금융/소비재) 외 종목입니다. 분석은 가능하나 정확도 보장 안 됨" 경고
- **쿼리에 욕설/PII 포함** → 가드레일 차단 + 재입력 안내

### E4. LLM 실패
- **LLM 호출 타임아웃 (>30초)** → Solar→GPT-4o-mini fallback 후 재시도. 재시도도 실패 시 "잠시 후 다시 시도해주세요" + 캐시된 직전 결과 노출
- **LLM 응답이 Pydantic 스키마 위반** → 1회 재시도 후 실패 시 해당 에이전트만 결과 비움. Tier 1은 부분 결론으로 출력 + 비완성 표시

### E5. 비용 임계
- **월 LLM 비용이 4만원 도달** → 모든 신규 분석은 캐시 모드 강제. 관리자 알림
- **월 5만원 도달** → 신규 분석 차단. "이번 달 사용량 초과" 안내. 관리자 검토 후 재개

### E6. 경계 사례
- **권유가 BUY인데 신뢰도 60% 미만** → 자동 HOLD 다운그레이드 + "근거 부족" 명시
- **권유가 BUY인데 포트 적합도 ★ (1점)** → HOLD 강제 + "내 포트엔 안 맞음" 명시
- **모든 시나리오에서 손실 예상** → "이 종목은 매수 비추천" 명확히 표시 (애매한 HOLD로 흐리지 않음)
