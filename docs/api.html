stock-agent 인터페이스 / API 명세

이 시스템은 REST 서버가 아니라 LangGraph 11노드 인터페이스 + MCP Tool + 산출물 인터페이스로 구성됩니다. 정본 문서는 docs/interface/interface_spec.md, 계약 스키마는 src/stock_agent/schemas/analysis.py, 노드 기준은 docs/architecture/pipeline_11node_groundtruth.md.

변경 안내: 기존 api.html은 빈 리다이렉트였습니다(강사 항목4 지적). 현재는 인터페이스 계약 실내용을 담은 정본 페이지입니다.

§1내부 노드 인터페이스 (11노드)

모든 노드는 AgentState(Pydantic)를 읽고 일부 필드를 채워 반환합니다. 시그니처는 전부 run_*(state: AgentState) -> AgentState.

#노드읽는 필드(입력)쓰는 필드(출력)출력 스키마

근거: graph/pipeline.py의 각 _node·_patch_from_agent.

§2라우팅 계약 (graph_route)

classifier가 산출하며 fan-out 워커 구성을 결정합니다.

{
  // classifier 산출 (pipeline.py:140-151)
  "analysis_scope": "single_stock | portfolio | sector",
  "urgency_reason": "surge | drop | earnings | news | general | null",
  "requested_depth": "summary | standard | deep",
  "summary_only": true,
  "worker_plan": ["quant", "qual", "competitor", "macro?"]  // macro는 조건부
}

§3MCP Tool 인터페이스

서버 FastMCP("stock-agent-peer-data") · stdio · python -m stock_agent.mcp_bridge.peer_data_server. Competitor 폴백 경로가 소비.

Tool입력출력
sector_rostersector: strlist[{stock_code, corp_name}]
market_metricsstock_codes: list[str], base_date: str=""record 목록 (아래)
 
{
  // market_metrics record (peer_data_server.py:46-77)
  "stock_code": "000660", "corp_name": "SK하이닉스", "base_date": "2026-05-21",
  "close_price": 201000, "market_cap": 146000000000000,
  "per": 12.3, "pbr": 1.8, "roe": 0.14,   // 0 이하·결측은 null
  "eps": 16300, "bps": 110000             // roe = EPS/BPS 근사
}

장애 계약: mcp/pykrx 미설치·조회 실패 → RuntimeError → 상위 mock 폴백. 핸드셰이크 검증: tests/mcp_bridge/test_mcp_handshake.py.
