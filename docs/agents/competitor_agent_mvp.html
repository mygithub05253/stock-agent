Competitor Agent MVP 요약

팀 회의용 빠른 확인 문서 · 2026-05-27

Competitor Agent는 국내 같은 섹터 peer를 선정하고, 대상 종목의 상대적 위치와 데이터 품질 경고를 계산해 Strategist가 사용할 수 있는 근거를 반환합니다. MVP 계산은 LLM 없이 Python/tool 결과로 수행합니다.

입력 Curator 결과의 stock_code, corp_code, sector
DB company, stock_price, financial_statement
출력 CompetitorResult: score, peer 목록, evidence, warnings

핵심 원칙

  • 국내 같은 섹터 peer만 사용하고 글로벌 peer는 임의로 추가하지 않습니다.
  • PER, PBR, ROE, 매출 성장률, 영업이익률, 부채비율은 peer_tool.py 계산 결과만 사용합니다.
  • 결측 지표는 결측으로 남기며 수치를 임의로 채우지 않습니다.
  • 점수는 투자 권유가 아니라 peer 대비 상대 위치를 설명하는 보조 지표입니다.

동작 흐름

  1. Curator가 대상 종목과 섹터를 확정합니다.
  2. run_competitor가 DB 연결 후 build_peer_comparison을 호출합니다.
  3. company에서 대상 기업과 같은 섹터 peer 후보를 조회합니다.
  4. stock_price와 financial_statement에서 최신 가격과 최근 재무 snapshot을 조회합니다.
  5. PER/PBR/ROE/성장률/마진/부채비율과 데이터 품질 점수를 계산합니다.
  6. 데이터 품질과 시가총액 근접도 기준으로 최종 peer를 선택합니다.
  7. 상대 위치, score, evidence, warnings를 CompetitorResult로 반환합니다.

경고 해석

  • peer_count_below_minimum: peer가 3개 미만이라 비교 신뢰도가 낮습니다.
  • sector_missing: 섹터 정보가 없어 같은 섹터 peer 선정이 제한되었습니다.
  • target_data_quality_low: 대상 종목의 핵심 데이터가 부족해 점수를 보수적으로 봐야 합니다.
  • per_not_applicable, pbr_not_applicable, roe_missing 등: 해당 지표를 계산할 수 없어 결측으로 처리했습니다.
  • mock_data_fallback: DB 실패 시 데모 흐름 유지를 위해 반환된 대체 결과이며 실제 DB 기반 비교가 아닙니다.

UI에서는 score보다 warnings와 data quality를 먼저 보여주는 것이 안전합니다. 특히 fallback 결과는 별도 배지로 명확히 표시해야 합니다.
