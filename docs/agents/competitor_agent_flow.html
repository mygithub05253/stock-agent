입력 — AgentState

Phase 1 — Rule-Based Peer 계산 ₩0

  • DB: company 테이블 → 같은 섹터 peer 후보 조회
  • 시가총액 밴드 필터: 0.25x ~ 4x (A3 명세 same_sector_marketcap_near)
  • DB: stock_price → 최신 시가총액, 종가
  • DB: financial_statement → 최근 3년 재무
  • 지표 계산: PER / PBR / ROE / 매출성장률 / 영업이익률 / 부채비율
  • Outlier 플래그: peer 중앙값 10배 초과 지표에 outlier_xxx 추가
  • 상대 위치: 지표별 백분위 → 0~100점 score (score=50이 peer 중립)
DB 연결 실패 → _mock_fallback_result() — mock_data_fallback 경고 포함, 파이프라인 계속
↓

Phase 2 — 24h 인메모리 캐시 확인 캐시

  • 키: {stock_code}_{YYYY-MM-DD} (모듈 레벨 dict)
  • 캐시 HIT → Phase 3 건너뜀, 저장된 narrative 재사용 → ₩0
  • 캐시 MISS → Phase 3 진행
  • Streamlit 재시작 시 캐시 초기화 (in-memory)
↓
(캐시 미스 AND OPENROUTER_API_KEY 설정된 경우에만)

Phase 3 — OpenRouter LLM 서술 생성 ~₩0.3/call

  • 모델: google/gemini-flash-1.5 (OPENROUTER_MODEL 환경변수로 변경 가능)
  • 프롬프트: prompts/competitor/system.md (GIC v11 Block J / G / F / B)
  • 입력: Phase 1 수치 JSON만 전달 — LLM이 숫자를 만들지 않음 (Block J)
  • 출력 JSON: peer_summary + evidence_cards + bear_case + data_gaps
  • OPENROUTER_API_KEY 없음 → 건너뜀 (₩0, rule-based 그대로)
LLM 실패 / 타임아웃 / key 없음 → None 반환, rule-based build_peer_summary() 유지
↓

출력 — CompetitorResult

  • score (0~100) — rule-based 백분위 계산
  • peer_summary — LLM이 있으면 향상된 서술, 없으면 rule-based 문자열
  • peers: list[dict] — 종목코드/이름/PER/PBR/ROE/마진/성장률 (rule-based)
  • evidence: list[str] — rule-based 근거
  • evidence_cards: list[dict] — LLM Evidence Card (finding/confidence/flag)
  • bear_case: str — LLM Short-seller 관점 (Block F)
  • data_quality_flags, warnings — rule-based 플래그
  • a1_peer_multiple_payload: Peer PER/PBR 중앙값 → A1 밸류에이션 연계
↓

다음 단계 → Strategist Agent

  • score, peer_summary, evidence를 Quant + Qual 결과와 종합
  • InvestmentAnalyst (GLM) → Guardrail 순으로 파이프라인 계속
  • Tier 2 Peer 탭: peer 메트릭 테이블 + evidence 카드 + bear_case 박스 렌더링

🛠 팀원 여러분께 — 이 에이전트는 파이프라인 중간 단계입니다. 자유롭게 개선해 주세요!

Competitor Agent는 Curator → Competitor → Strategist 흐름의 중간에 위치합니다. AgentState.competitor (CompetitorResult)의 기존 필드를 유지하는 한, 내부 로직은 언제든지 수정·개선할 수 있습니다. 수정하다 변경이 필요하면 자유롭게 건드려도 됩니다.

수정하고 싶은 것파일방법
LLM 프롬프트 품질 개선 prompts/competitor/system.md 마크다운 직접 수정 — Python 코드 변경 불필요
LLM 모델 교체 .env OPENROUTER_MODEL=원하는_모델 한 줄만 변경
Peer 선정 기준 변경 tools/peer_tool.py select_peer_rows() 또는 밴드 상수 수정
점수 가중치 조정 tools/peer_tool.py calculate_relative_position()의 weight 수정
UI 레이아웃 변경 streamlit_app.py _render_peer_tab() 함수만 수정
새 지표 추가 (EV/EBITDA 등) tools/peer_tool.py + schemas/analysis.py PeerMetricRow 필드 추가 + calculate_metric_row 수정

피드백이나 개선 제안은 GitHub Issues 또는 단톡방 (#stock-agent-pm) 으로 알려주세요!
