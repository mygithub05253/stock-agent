1. 발표용 전체 동작 흐름

왼쪽은 데이터 준비 단계, 오른쪽은 실제 Agent 실행 단계입니다. 핵심은 뉴스가 DB에 저장되는 것에서 끝나지 않고, Qual Agent의 evidence와 risk로 변환된다는 점입니다.

Collect

뉴스 수집

news_collector.py가 종목별 뉴스를 수집하고 제목, 요약, URL, 발행일을 정리합니다.

Store

원천 저장

raw_news에는 재처리 가능한 원본을, rag_documents에는 검색용 문서를 저장합니다.

Embed

청크 임베딩

embed_news.py가 문서를 chunk로 나누고 BAAI/bge-m3 임베딩을 만듭니다.

Retrieve

Hybrid 검색

pgvector 의미 검색과 키워드 검색을 함께 수행하고 RRF로 순위를 합칩니다.

Rerank

선택형 재정렬

RAG_RERANKER_ENABLED가 켜지면 CrossEncoder가 후보 근거를 재정렬합니다.

Qual

정성 분석

qual.py가 검색 결과를 evidence와 risk로 바꾸고 sentiment와 score를 계산합니다.

Output

전략 반영

QualResult가 Strategist와 최종 분석 결과에 들어가 사용자에게 근거로 표시됩니다.

2. 구현상 중요한 점

의미 검색

단순 regex가 아니라 chunk embedding을 저장해 의미적으로 가까운 뉴스를 찾습니다.

Hybrid/RRF

임베딩만 믿지 않고 키워드 랭킹도 합쳐 금융 기사 제목/키워드 누락을 줄였습니다.

Fallback

DB나 검색 실패 시 Agent가 죽지 않고 fallback evidence와 이유를 남깁니다.

4. 팀장님 발표용 핵심 멘트

김도예님 파트의 핵심은 뉴스 데이터를 단순히 모으는 데서 끝내지 않고, Qual Agent가 실제 분석 근거로 사용할 수 있도록 RAG 저장, 임베딩, Hybrid 검색, fallback까지 연결했다는 점입니다. 검색 자체는 Hit@5와 MRR 기준으로 안정적으로 동작했고, 남은 과제는 최종 생성 문장이 검색 근거를 더 충실히 인용하도록 만드는 것입니다.
