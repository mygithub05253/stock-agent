진행해야 할 것

카드를 클릭하면 세부 액션이 펼쳐집니다.

  • DART: 대상 기업 범위(MVP 9종목 vs KOSPI200), 재무 항목(손익·재무상태·현금흐름), 기간(예: 최근 3~5년), 공시 유형(정기·주요사항)을 정한다.
  • 매크로(ECOS): 어떤 지표(기준금리·환율·CPI·국채금리 등)와 주기(일/월)를 적재할지 정한다.
  • 수집 범위 → raw_news/raw_macro/financial_statement 적재량과 임베딩 비용에 직결되므로 비용 상한과 함께 결정.
  • 합의안을 docs/decisions에 ADR로 짧게 기록.
  • Postgres가 vector 기능을 제공하므로 Supabase 1대(pgvector)로 MVP를 끝낼 수 있음 — ADR-001과 동일한 방향.
  • 다만 벡터DB 사용 경험을 위해 Chroma 등 별도 벡터DB 1개를 병행해 보는 것도 학습 가치가 있음.
  • 결정 포인트: 운영은 pgvector 단일로 가되, Chroma는 optional backend로 인터페이스만 분리해 실험용으로 남길지.
  • src/stock_agent/rag/pgvector_store.py 인터페이스를 추상화해 백엔드 교체 가능하게 설계.
  • OpenRouter 계정 생성 → 소액 결제 → API 키 발급.
  • 키는 .env에만, .env.example엔 키 이름만 추가(협업 규칙).
  • 하나의 엔드포인트로 GPT·Claude·Solar 등 여러 모델을 골고루 사용 → src/stock_agent/llm 라우팅과 연결.
  • 월 5만원 상한(ADR-004) 안에서 단순 작업=작은 모델, 복잡 결정=큰 모델 라우팅 유지.
