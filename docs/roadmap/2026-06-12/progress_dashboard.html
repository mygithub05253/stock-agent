Instructor Feedback

지금 가장 집중해야 할 피드백

강사님이 명시한 우선순위와 가중치 효과를 기준으로, 팀 작업 순서를 다시 압축했습니다.

P0 Qual 폴백 구멍부터 닫기

qual.py는 retriever 실패를 잡지 않아 DB·임베딩 환경 문제를 전체 파이프라인 크래시로 전파합니다.

  • retriever 호출 try/except + 빈 결과 fallback
  • CI 제외 테스트 3건 재활성화
  • 담당: doyekeem 중심, 윤수정 협업

P0 오픈웨이트 sLLM 실호출

현재 OpenRouter 기본 모델은 openai/gpt-4o-mini라 항목 #3의 sLLM 요구를 충족하지 못합니다.

  • 300B 이하 오픈웨이트 모델을 기본 또는 검증 노드에 연결
  • Guardrail 게이팅·관측과 묶어 #3 가중점 회복
  • 담당: 백형준 중심, 이동원 설정·평가 지원

P1 LangGraph StateGraph 전환

역할별 파일은 분리돼 있지만 실행은 여전히 순차 함수 호출입니다. Multi-Agent 차별성이 가장 크게 막힌 지점입니다.

  • worker fan-out + Strategist join + 조건부 라우팅
  • Macro Agent도 실행 경로에 포함
  • 담당: 백형준, melinamuun 협업

P1 RAGAS 지표 확장과 실데이터 재측정

RAGAS는 공식 4점을 받았지만 context precision/recall과 키 없는 재실행 증거가 부족합니다.

  • context_precision·context_recall·relevancy 포함
  • Qual RAG 수정 후 누적 리포트 생성
  • 담당: 이동원, doyekeem 협업

P2 Hybrid + Reranker 실연결

DB의 tsvector GIN 인덱스는 준비됐지만 검색 코드는 pgvector 단독입니다.

  • 키워드 + 벡터 RRF 결합
  • reranker와 청킹 추가
  • 담당: doyekeem

P2 Strategist 변별력과 Guardrail 게이팅

골든셋이 전부 HOLD이고 Guardrail은 passed=True 고정이라 최종 판단 품질을 증명하기 어렵습니다.

  • Macro 포함 가중 종합과 부분 실패 허용
  • passed=False + 재합성 경로
  • 담당: 백형준, melinamuun 협업
