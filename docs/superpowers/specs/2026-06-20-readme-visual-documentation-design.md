# README 및 시각 문서화 설계

## 목표

처음 방문한 사용자가 프로젝트의 목적, 실행 방법, 시스템 구조, 검증 성과를 루트 `README.md`에서 빠르게 이해하고, 각 책임 폴더의 작업자가 해당 영역의 역할과 동작 원리를 독립적으로 파악할 수 있게 한다.

## 범위

- 루트 `README.md`에 프로젝트명과 한 줄 설명, 배지, 썸네일, 프로젝트 소개, 기술 스택, 실행 방법, 아키텍처 이미지, 성과 지표, 디렉토리 구조를 보강한다.
- 기존 협업 가이드, 데이터 저장 규칙, LLM 비용 정책, 문서 영역, Docker 명령, 변경 이력은 삭제하지 않고 현재 구조에 맞게 유지한다.
- 숨김 폴더, 캐시, 빌드 산출물, Git worktree를 제외한 책임 단위 폴더에 `README.md`를 둔다.
- 실행 코드가 있는 폴더는 상세 설명을, 문서·자료 보관 폴더는 간결한 인덱스를 제공한다.
- Streamlit UI 코드는 변경하지 않고 현재 화면을 바탕으로 개선 제안 문서를 작성한다.

## 문서 계층

### 루트 README

루트 문서는 다음 순서로 구성한다.

1. 프로젝트명, 한 줄 설명, 배지, 썸네일
2. 프로젝트 소개: What, Why, 핵심 기능 3줄
3. 기술 스택 표
4. 빠른 실행: clone, 가상환경, 의존성 설치, Streamlit 실행
5. 시스템 아키텍처 이미지와 상세 문서 링크
6. 검증 가능한 주요 결과와 성과 지표
7. 디렉토리 구조
8. 기존 용어집, 구현 상태, 협업, 저장, 비용, 문서, Docker, 변경 이력

### 폴더 README

폴더 성격에 따라 두 등급으로 작성한다.

- 상세형: `datas/`, `db/`, `eval/`, `src/stock_agent/`, `tests/`, `ui/` 및 실행 책임이 있는 하위 폴더. 한 줄 설명, What/Why, 핵심 기능, 기술 스택, 동작 원리, 구조, 성과 또는 검증 방법, 디렉토리 구조를 포함한다.
- 인덱스형: `docs/`의 주제별 보관 폴더, `notebooks/`, `scripts/`, 프롬프트 하위 폴더 등. 폴더 목적, 주요 파일, 사용·갱신 규칙, 관련 링크를 포함한다.

모든 수치와 구현 상태는 코드, 테스트, 평가 리포트에서 확인되는 사실만 사용한다. 계획 상태는 현재 구현과 명확히 구분한다.

## 시각 자산

### 프로젝트 썸네일

- 실행 중인 `http://localhost:8501`의 데스크톱 화면을 캡처해 UI 구조와 색상을 확인한다.
- GPT Image 2.0 기본 이미지 생성 경로로 한국 주식 멀티 에이전트 분석 제품을 표현하는 가로형 대표 이미지를 만든다.
- 이미지 내부의 긴 문장은 피하고, README 본문에서 프로젝트명과 설명을 텍스트로 제공한다.
- 최종 파일은 `docs/assets/readme/stock-agent-thumbnail.png`에 저장한다.

### 시스템 아키텍처

- 코드와 최신 문서를 바탕으로 현재 동작 경로를 Mermaid로 먼저 정의한다.
- 흐름은 `Streamlit -> Intake/Curator -> 전문 Agent -> Strategist/InvestmentAnalyst -> Guardrail/Renderer`와 외부 데이터·Postgres/pgvector 연결을 보여준다.
- Mermaid 원본은 정확성의 기준으로 저장하고, 이를 시각 참조로 GPT Image 2.0에 제공해 README용 가로형 인포그래픽을 만든다.
- 생성 이미지의 라벨이 부정확하면 반복 생성하거나 긴 라벨을 줄인다. 정확한 세부 구조는 Mermaid 원본 링크로 보완한다.
- 최종 파일은 `docs/assets/readme/stock-agent-architecture.png`에 저장한다.

## 성과 지표

성과 표에는 저장소에서 재현 가능한 항목만 싣는다.

- Phase 1 평가: 규칙 기반 검증 40/41
- Competitor 회귀 평가: 6/6
- 골든셋 평가 케이스: 5개
- 온보딩 질문 카드: 7단계
- 분석 진행 Agent: 코드의 진행 순서 기준 9개
- RAGAS faithfulness 0.4096은 목표 0.80 미달임을 함께 표기해 과장하지 않는다.

## Streamlit UI 개선 제안

별도 문서에서 현재 첫 화면과 결과 화면의 문제를 우선순위로 정리한다.

- 첫 화면에 제품 가치와 진행 단계를 더 명확히 표시
- 질문 카드를 좁은 읽기 폭의 컨테이너로 묶고 하단 행동 버튼 정렬 개선
- 기본 선택의 자동 확정처럼 보이는 인상을 줄이도록 선택 상태 처리 개선
- 결과 화면의 Tier 1 핵심 판단을 첫 viewport에 집중
- Agent 진행 상태와 근거 탭의 정보 위계 개선
- Streamlit 제약 안에서 CSS와 기본 컴포넌트를 최소한으로 조합하는 구현 방향 제시

## 검증

- 모든 관리 대상 폴더에 README가 있는지 스크립트로 검사한다.
- 루트 README의 로컬 이미지와 문서 링크가 실제 파일을 가리키는지 검사한다.
- clone 및 실행 명령이 현재 저장소 설정과 일치하는지 확인한다.
- `pytest`를 실행해 문서 작업이 기존 기능을 손상하지 않았는지 확인한다.
- 데스크톱과 모바일 viewport에서 Streamlit을 열어 페이지 정체성, 빈 화면, 오류 오버레이, 콘솔 경고·오류, 핵심 상호작용을 확인한다.
- 생성 이미지를 직접 열어 텍스트 정확성, 잘림, 워터마크, 프로젝트 적합성을 확인한다.
