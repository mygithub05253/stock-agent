# 최종 발표 자료 설계

## 목표

6월 27일 최종 발표회에서 1조 stock-agent를 15분 안에 설득력 있게 설명할 수 있는 HTML 슬라이드와 PPTX를 만든다. 발표의 중심은 "개인투자자의 구조적 의사결정 문제를 멀티에이전트 구조로 어떻게 완화했는가"이며, 시장조사, Agent 활용 이점, 실제 구현, 검증 결과, 한계와 발전 가능성을 균형 있게 보여준다.

## 발표 방향

선택한 방향은 **심사 설득형**이다.

- 문제 정의와 시장 수요를 3~4장으로 압축한다.
- 중반부는 실제 stock-agent의 데모 흐름, 시스템 구성도, 11노드 Agent 흐름으로 설득한다.
- 후반부는 검증 수치, 상업성, 한계, 팀원 역할로 마무리한다.
- 발표 본문에는 출처를 크게 노출하지 않되, speaker note 또는 백업 슬라이드에는 출처와 기준 시점을 남긴다.

## 리서치 슬라이드 압축 원칙

`stock_agent_research.html`의 10장은 그대로 이어 붙이지 않는다. 최종 자료에서는 다음처럼 압축한다.

- 원본 2장, 3장, 7장의 핵심 수치는 최종 2~3장으로 병합한다.
- 원본 4장은 최종 3장 문제 정의로 재작성한다.
- 원본 5장은 최종 5장 "왜 멀티 Agent인가"로 프로젝트 구현 용어에 맞게 교정한다.
- 원본 6장은 최종 4장 경쟁 대안 표로 사용하되 "stock-agent (제안)" 대신 "stock-agent (구현)"으로 표현한다.
- 원본 8장은 차트 아이디어만 채택하고, 최종 자료에는 실제 차트 1~2개만 넣는다.
- 원본 9장 출처 목록은 본문에서 제외하고 speaker note 또는 백업 슬라이드로 보낸다.

## 수치와 표현 정합

발표에서 사용할 수치는 다음처럼 방어 가능한 표현으로 통일한다.

- 개인투자자 수: **1,442만 명**. 2025년 말 개인투자자 기준으로 사용한다.
- 전체 주식소유자/실질주주 수: **1,456만 명**. 개인투자자 수와 섞지 않는다.
- 주식거래 활동계좌: **1억 500만 개**. 2026년 4월 29일 금융투자협회 기준으로 사용한다.
- 개인 MTS 거래비중: **65.7%**. 2026년 1~5월 한국거래소 보도 기준으로 사용한다.
- 개인 평균 수익률: **-16.77%**. 공식통계가 아니라 NH투자증권·머니투데이 표본 분석임을 note에 남긴다.
- 1~3종목 집중투자: **59.7%**. 한국예탁결제원 기반 보도 기준으로 사용한다.
- 디지털 금융이해력: **59.3점/100점**. 한국금융소비자보호재단 2025년 조사 기준으로 사용한다.
- MTS 투자자 초과수익률: **-5.4%**. 자본시장연구원 김준석(2022)의 2020년 3~10월 분석 기준으로 사용한다.

현재 구현과 어긋날 수 있는 표현은 피한다.

- "악마의 대변인 에이전트"라고 하지 않는다. 실제 구현 기준으로는 **Guardrail + 분리된 전문 Agent 관점 검증**이라고 설명한다.
- "기술적·펀더멘털·리스크·센티먼트" 대신 **정량·정성·Peer·거시·안전검증**으로 통일한다.
- "투자 추천"보다 **분석 신호**, **의사결정 보조**, **투자 권유 아님**을 사용한다.

## 시각 스타일

최종 자료는 현재 리서치 HTML의 어두운 금융 리포트 스타일을 참고하되, 실제 프로젝트 화면과 README 자산이 돋보이도록 조정한다.

- 색상: 짙은 남색 배경, 파란색/청록색 강조, 위험 수치는 빨간색, 검증 성과는 초록색.
- 톤: 투자 리포트처럼 신뢰감 있게, 하지만 과도한 애니메이션과 장식은 줄인다.
- 제목: 한 장당 하나의 메시지로 짧게 작성한다.
- 본문: 발표자가 말로 설명할 수 있도록 슬라이드 텍스트는 3~5개 포인트로 제한한다.
- 표: 경쟁 대안, 팀 역할, 검증 결과에만 사용한다.
- 차트: 시장 수치와 검증 결과에 1~2개만 사용한다.

## 발표 시간 배분

총 15분 발표 기준 권장 시간은 다음과 같다.

- 1장: 30초
- 2~5장 문제·시장·Agent 필요성: 4분
- 6장 데모 시나리오: 1분 30초
- 7~10장 구현·Agent·차별점: 5분
- 11장 검증 결과: 1분 30초
- 12~13장 상업성·한계: 1분 30초
- 14장 팀 역할·마무리: 1분

Q&A에서 깊게 받을 질문은 RAGAS 0.41, 투자 권유가 아닌 이유, 실데이터 연결 범위, 상업화 시 규제/책임 범위, 기존 증권앱과의 차이로 예상한다.
