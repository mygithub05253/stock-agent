## 교차검증 결과 (문서 주장 ↔ 코드 실태)

코드 우선 원칙으로 다음 불일치를 확인, 항목4에 반영:

1. **ERD 테이블명 불일치**: `erd.md`는 `financial_data`/`disclosure`로 명시하나 실제 `db/init/001_...sql`은 `financial_statement`/`disclosure_report`+`disclosure_content`. 코드(`financial_tool.py:14,28,81`, `rag_tool.py:11-12`)도 실제 테이블명 사용 → ERD가 구형 설계안.
2. **미구현 테이블**: ERD의 `users`/`holdings`/`analysis_history`는 DDL에 없음. 런타임 인메모리 Pydantic 객체만 존재.
3. **시퀀스 다이어그램 ↔ 파이프라인**: 다이어그램은 4단계 병렬, 코드는 8단계 순차(`request_classifier`·`investment_analyst` 미도식). 항목1·10의 "순차·비병렬" findings와 일치.
4. **관측 선언 vs 실동작**: README가 LangSmith "준비 중, placeholder만"이라 자인 → 항목3 (c) 미동작 판정과 일치, 과대선언 없음.
