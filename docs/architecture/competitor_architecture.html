Pipeline Position

파이프라인 내 위치

Curator가 종목을 확정하면 Quant·Qual과 함께 Phase 1 워커로 실행되고, 결과 CompetitorResult는 Strategist 종합에 소비됩니다.

InvestorProfile→ Curator→ RequestClassifier→ Quant · Qual+ Competitor→ Strategist→ InvestmentAnalyst→ Guardrail

Fallback Strategy

데이터 가용성 3단 폴백

데이터 소스가 실패해도 데모 흐름이 끊기지 않도록 우선순위를 둡니다. 실데이터(①②)와 mock(③)을 데이터 품질 플래그로 정직하게 구분합니다.

DB

① Postgres 실데이터 실데이터

build_peer_comparison(conn, …) — company·stock_price·financial_statement에서 PER/PBR/ROE/성장률/마진/부채비율 계산. 기본 경로.

MCP

② MCP 실시간 시세 실데이터·라이브

DB 실패 시 자체 FastMCP 서버를 stdio 자식 프로세스로 띄워 pykrx 시세 기반 peer 비교. mcp_live_market_data 마커 부여. build_comparison_from_market_rows가 동일 점수 엔진 재사용.

MOCK

③ 하드코딩 mock 데모 최후 보루

MCP까지 불가하면 mock_data_fallback 반환. 데모 중단 방지용이며 실제 투자 분석값이 아님을 data_quality_flags로 명시.

실데이터(MCP)는 "mock"/"fallback"/"데모용" 문자열을 의도적으로 배제해, Guardrail mock_data_audit가 실데이터를 mock으로 오판하지 않게 합니다.

Peer Selection Engine

복합 유사도 peer 선정 (#62)

데이터 완성도만 1순위로 보던 한계를, 시총 band 1차 거름 후 복합 유사도 정렬로 개선했습니다. 결측 지표는 가용 항목으로 가중치를 재정규화합니다.

시가총액 근접(규모)
0.45
영업이익률(사업 경제성)
0.30
데이터 완성도
0.25

1차 거름: 시총 band 0.25x~4x (내 시총의 ¼~4배). 이상치(중앙값 10배 초과)는 outlier_* 플래그. 종합 score = 밸류 20 · 수익성 25 · 성장 20 · 마진 15 · 재무안정 10 · 완성도 10 (0~100).

MCP / A2A

MCP 외부 노출 (루브릭 #6)

Competitor 폴백 전용이 아니라, 다른 에이전트·외부 MCP 클라이언트가 소비할 수 있는 표준 stdio 서버로 노출합니다.
