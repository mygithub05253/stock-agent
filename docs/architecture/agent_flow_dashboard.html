Pipeline

전체 실행 흐름 (build_analysis_graph · graph.stream)

코드의 실제 실행 순서입니다. classifier의 worker_plan에 따라 4개 워커가 Send로 병렬 fan-out되고 Strategist로 join됩니다(macro는 조건부). 각 노드 클릭 → 아래 상세 패널 이동.

0 InvestorProfile 투자성향 구조화온보딩 답변 → UserProfile (UI 접수 단계, 파이프라인 진입 전) 사전 단계
↓
1 Curator 분석 대상 큐레이션질문+포트폴리오 → 종목코드·섹터·corp_code 확정 run_curator
↓
2 RequestClassifier 의도 분류질문 → intent·scope·urgency 구조화 run_request_classifier
↓
⑂ Worker Fan-out · Quant·Qual·Competitor·Macro (Curator 출력만 의존 → 병렬화 가능)
3 Quant재무·시세 정량
4 Qual뉴스·공시 RAG
5 CompetitorPeer 횡비교
6 Macro거시경제
↓
7 Strategist 가중 종합4워커 가중 합성(Quant .40·Qual .25·Comp .20·Macro .15) + 부분실패 허용 run_strategist
↓
8 InvestmentAnalyst GLM 보정최종 분석 신호·포트폴리오 적합도 보정 (GLM → rule-based fallback) run_investment_analyst
↓
9 Guardrail 안전 게이팅7게이트 검증 → passed=not blocked + 재합성 rewrite 루프 run_guardrail
↓
✓ Output Tier 1/2/3신호+신뢰도+근거 카드 → Streamlit UI 렌더 AnalysisOutput

💡 노드를 클릭하면 아래 상세가 해당 에이전트로 바뀝니다.

Resilience

폴백 전략 — "데모는 절대 멈추지 않는다"

강사 재검토에서 강점으로 인정된 축입니다. 모든 워커가 DB·LLM 실패에 안전하게 degrade 합니다.

① Competitor 3단계

가장 정교한 폴백 체인.

  • DB peer_tool 정상 경로
  • 실패 → MCP pykrx 실시간 시세(실데이터)
  • 그래도 실패 → mock (최후 보루)

② Qual / Quant

retriever·DB 호출을 try/except로 감쌈(#52·#53).

  • Qual: retrieve_*_with_fallback
  • Quant: _connect_with_retry + 기본값
  • 크래시 전파 차단

③ Strategist / Guardrail

부분 실패 허용 + 안전 게이팅.

  • Strategist: 워커 결손 시 가중 재정규화(#51)
  • Guardrail: 7게이트 → 차단·재합성(#50)
  • LLM 실패 → 보수적 결과

Now vs Target

현재 구조 vs 목표 구조 (정직하게)

발표 시 "어디까지 했고 다음이 무엇인지"를 솔직히 보여주는 슬라이드입니다.

달성 LangGraph StateGraph 11노드

  • build_analysis_graph()가 11노드 그래프를 컴파일, graph.stream()으로 라이브 실행

  • RequestClassifier worker_plan으로 conditional 라우팅 + macro 조건부 실행

  • 역할 분리·폴백·게이팅·MCP·Macro 가중·revision loop 모두 동작

잔여 폴리시 고도화
