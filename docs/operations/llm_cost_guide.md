## 0. 한 줄 결론

> **부트캠프 무료 크레딧(Solar) + 모델 라우팅 + 응답 캐싱** 세 가지로 월 5만원 상한 안에서 충분히 운영 가능. "에이전트 6개라서 비싸진다"는 *숫자상 사실이 아닙니다*.

## 1. 왜 이 문서가 필요한가요?

팀원들이 *"에이전트가 6개면 LLM 호출이 6배니까 비용이 6배 들지 않을까?"* 하고 걱정하셨습니다. 이 문서는 그 걱정에 **숫자로 답하고**, 실제로 비용을 낮추는 방법 6가지를 설명합니다.

## 2. 1회 분석 비용 — 실제 계산 (쉬운 설명)

### 2.1 한 번 분석하면 LLM이 몇 번 호출되나?

박민호 페르소나가 "삼성전자 어떻게 할까?" 한 번 물으면, 6 에이전트가 돌아가면서 LLM을 호출합니다:

| 에이전트 | 호출 횟수 | 무엇을 시키나? |
|----------|-----------|----------------|
| Curator | 2회 | ① 자연어에서 종목·의도 파싱 ② (종목 미지정 시) 후보 추천 |
| Qual Worker ★ | 3회 | ① 뉴스 RAG 검색 ② 호재/악재 요약 ③ 감성 분석 |
| Quant Worker | 1회 | 5y 가정 설명문 (수치 계산은 Python으로 — LLM 거의 안 씀) |
| Competitor | 2회 | ① Peer 추출 ② 비교 분석 |
| Strategist | 3회 | ① 4 결과 종합 ② 자체 검증 ③ PB 리포트 작성 |
| Guardrail | 2회 | ① 출력 필터 ② RAGAS 채점 |
| **합계** | **~13회** | |

### 2.2 1회 호출에 드는 토큰

> **토큰**: LLM이 글을 읽고 쓰는 단위. 한국어 기준 1글자 ≈ 1.5~2 토큰.
> 예) "안녕하세요" = 약 7 토큰

평균적으로 1회 호출당:
- **입력 (Input)**: 5,000 토큰 (시스템 프롬프트 + 검색된 RAG 컨텍스트)
- **출력 (Output)**: 1,000 토큰 (요약 또는 결정)

→ 1회 분석 = 13 × 6,000 = **78,000 토큰** (10만 토큰 미만)

### 2.3 모델별 1회 분석 비용

| 모델 | 1회 분석 | 발표 데모 30회 | 12주간 1,000회 | 비고 |
|------|----------|----------------|------------------|------|
| **Solar (Upstage)** | **0원** ⭐ | 0원 | 0원 | 부트캠프 무료 크레딧 (수만 회 분량) |
| **gpt-4o-mini** | ~36원 | ~1,080원 | ~36,000원 | OpenAI 가장 저렴 모델 |
| Claude Haiku 3.5 | ~80원 | ~2,400원 | ~80,000원 | Anthropic 저렴 모델 |
| gpt-4o | ~600원 | ~18,000원 | ~600,000원 | 비싸지만 고품질 |

→ **Solar 무료 크레딧 + gpt-4o-mini fallback 조합** 이면 12주 전체에서 **5만원 미만** 으로 끝납니다.

### 2.4 비용이 폭발하는 시나리오 (피해야 할 패턴)

❌ **Self-Consistency N=10** — 같은 질문을 10번 반복해서 다수결. 비용 10배. 우리는 N=3 사용.
❌ **모든 호출을 gpt-4o로** — 단순 파싱까지 비싼 모델. 우리는 *모델 라우팅* 으로 회피.
❌ **캐싱 없음** — 같은 종목을 어제 분석했는데 또 호출. 우리는 24h 캐시 사용.
❌ **사용자가 무한정 분석 요청** — Phase 1 종료 후 Rate Limit (회원당 1일 10회) 적용 검토.

## 3. 비용 절감 6 패턴 — 쉬운 설명 + 우리 적용

### 3.1 패턴 1: 모델 라우팅 (Model Routing)

> **개념**: 작업의 난이도에 따라 작은/큰 모델을 골라 쓰는 것. 식당에서 라면은 가스불, 스테이크는 오븐 쓰듯.

우리 적용:
| 작업 | 모델 | 이유 |
|------|------|------|
| 자연어 의도 파싱 (Curator) | **gpt-4o-mini** 또는 Solar | 단순 작업 → 작은 모델로 충분 |
| 뉴스 요약 (Qual) | **Solar** | 한국어 강점 + 무료 크레딧 |
| 최종 결정 (Strategist) | **gpt-4o** 또는 Solar | 복잡한 종합 추론 → 큰 모델 |
| 가드레일 필터 (Guardrail) | **gpt-4o-mini** + 정규식 | 단순 패턴 매칭 |

구현 위치: `src/stock_agent/llm/factory.py` + `routing.py`

### 3.2 패턴 2: 응답 캐싱 (Response Cache)

> **개념**: 같은 질문이 또 들어오면, LLM에 다시 묻지 말고 *어제 답을 그대로 보여주기*. 카페에서 같은 메뉴 두 번 시킬 때 한 번에 두 잔 만드는 것과 같음.

우리 적용:
- **회원·종목·24시간** 키로 분석 결과 캐싱
- Postgres `analysis_cache` 테이블에 저장
- 24시간 안에 같은 회원이 같은 종목을 또 묻으면 → 캐시 반환 (LLM 호출 0회)
- 발표 시연용 **"캐시 모드 강제 토글"** 도 제공 (관객 앞에서 같은 시연 N번 반복할 때 비용 0원)

### 3.3 패턴 3: 프롬프트 캐싱 (Prompt Cache)

> **개념**: 시스템 프롬프트(긴 지시문)는 호출마다 똑같음. OpenAI/Claude가 *같은 프롬프트는 50% 할인* 해줌.

우리 적용:
- 모든 에이전트의 시스템 프롬프트 5분간 캐시 (OpenAI 기본 설정)
- 효과: 같은 사용자 세션 안에서 호출 비용 약 30~40% 절감

### 3.4 패턴 4: 배치 사전 분석 (Pre-compute)

> **개념**: 사용자 요청을 기다리지 않고, **새벽에 인기 종목을 미리 분석해두기**. 카페에서 출근 시간 전에 미리 만들어두는 콜드브루처럼.

우리 적용:
- 매일 새벽 3시: KOSPI 시총 상위 9종목(반도체·금융·소비재) 자동 분석
- 결과는 캐시에 저장
- 사용자가 그 종목 분석 요청 시 → 캐시 반환 (LLM 호출 0회)

### 3.5 패턴 5: Self-Consistency N 축소

> **개념**: AI가 같은 질문에 N번 답하게 하고 다수결로 결정. N이 높을수록 신뢰도 ↑ 비용 ↑.

우리 적용:
- 일반 작업: N=1 (한 번만 호출)
- Strategist 최종 결정만: **N=3** (5에서 축소)
- 효과: Strategist 단일 비용을 5/3로 절감

### 3.6 패턴 6: 계산은 Python으로 (LLM 안 씀)

> **개념**: PER·MoS·평가손익 같은 *수식 계산* 은 LLM이 아니라 Python 코드가 해야 함. LLM은 글 쓰는 데만 사용.

우리 적용:
- Quant Worker: DCF·MoS·평가손익 모두 Python 함수
- LLM은 *결과를 자연어로 설명* 할 때만 호출 (1회만)
- 효과: 정량 계산 비용 대폭 절감 + 정확도 ↑ (LLM이 숫자 틀리는 환각 방지)

## 4. 비용 모니터링 — 어떻게 추적하나?

### 4.1 LangSmith 대시보드 (추후 도입)

> **LangSmith**: LangChain/LangGraph 기반 LLM 호출의 *입출력·비용·시간* 을 추적하고 디버깅하는 도구.

8주차에 도입 예정. 도입 후 PM은 다음을 매주 점검:
- **이번 주 누적 비용** (대시보드 메인 카드)
- **에이전트별 비용 비중** (어떤 에이전트가 가장 비싼가?)
- **모델별 호출 분포** (Solar:GPT-4o-mini 비율)

### 4.2 비용 알림 (자동)

| 임계 | 동작 |
|------|------|
| 월 3만원 도달 | Slack 알림 — "이번 달 사용량 60%" |
| 월 4만원 도달 | 모든 신규 분석은 **캐시 모드 강제** |
| 월 5만원 도달 | **신규 분석 차단** + 관리자 검토 후 재개 |

### 4.3 Postgres 비용 로그 테이블 (예정)

```sql
CREATE TABLE llm_cost_log (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    user_id UUID,
    agent_name VARCHAR(50),
    model VARCHAR(50),
    input_tokens INT,
    output_tokens INT,
    cost_krw DECIMAL(10,2),
    cached BOOLEAN,
    timestamp TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

## 5. 자주 묻는 질문 (FAQ)

### Q1. 부트캠프 무료 크레딧이 정말 충분한가요?

A. Solar(Upstage) 무료 크레딧은 일반적으로 **수십만 토큰 ~ 수백만 토큰** 제공. 우리 1회 분석이 78K 토큰이니, **수백 ~ 수천 회 분석** 가능. 부트캠프 12주 + 발표 데모로는 충분.

### Q2. 모델이 갑자기 안 되면요? (장애)

A. **Fallback 체인** 구성: Solar → gpt-4o-mini → Claude Haiku → 캐시. 어떤 LLM이 죽어도 시스템은 계속 동작.

### Q3. Streamlit Cloud 발표 시연에서 라이브로 호출하면 비용이 폭증하지 않을까요?

A. 발표용 **캐시 모드** 를 만들어 라이브 호출 0회. 심사위원이 "라이브로 보여주세요" 하면 그때만 LIVE 토글 → 1회 분석 60원 미만.

### Q4. 만약 5만원 상한을 넘으면 어떻게 되나요?

A. PRD §7 E5에 명시된 대로:
- 4만원 도달 → 캐시 강제
- 5만원 도달 → 신규 분석 차단 + PM 검토 후 재개

### Q5. 회원이 너무 많아져서 비용이 폭증하면요?

A. MVP는 **회원 50명 이내** 가정. Phase 3에서 본격 회원이 늘어나면 별도 가격 정책(유료 plan)을 검토.
