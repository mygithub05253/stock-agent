Problem

정보는 많지만 근거 있는 통합 판단은 어렵다

  • 1~3종목 집중투자 비중 59.7%: 포트폴리오 리스크가 개인에게 집중

Agent Rationale

왜 멀티 Agent인가

  • 정량 Agent: DART·시세 기반 재무/밸류에이션 지표 계산
  • 정성 Agent: 뉴스·공시 RAG로 호재/악재 근거 탐색
  • Peer·Macro Agent: 같은 업종 비교와 거시 환경 보완
  • Guardrail: 투자 권유·위험 표현을 규칙 기반으로 완화

역할 분리의 이점

한 모델이 모든 판단을 말하는 구조보다, 계산·검색·합성·검증을 분리하는 구조가 더 설명 가능하고 실패 격리가 쉽습니다.

Architecture

UI, DB/RAG, LangGraph, 안전 검증으로 구성

  • Streamlit UI에서 온보딩과 결과 제공
  • PostgreSQL + pgvector로 정형 데이터와 RAG 검색 통합
  • LangGraph StateGraph와 Send API로 worker Agent 동적 fan-out
  • LLM은 해석·합성, Guardrail은 규칙 기반 검증
