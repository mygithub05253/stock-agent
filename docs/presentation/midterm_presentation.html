분석 지표

지표의미산식
PER주가 수익 배수주가 ÷ EPS
PBR순자산 배수주가 ÷ BPS
ROE자기자본 수익률순이익 ÷ 자본
부채비율재무 건전성총부채 ÷ 자본
매출성장률성장세YoY % 변화
영업이익률수익성영업이익 ÷ 매출
💡 Calculation Outside LLM 원칙
모든 수치 계산은 Python/DB에서 수행
LLM은 해석 텍스트 생성에만 사용
→ 정확성 보장 + 비용 절감

왜 멀티에이전트인가?

단일 LLM 방식의 한계

  • ❌ "왜 이 신호인가?" 추적 불가
  • ❌ 재무·뉴스·경쟁사 분석이 혼재
  • ❌ 성향에 따른 개인화 경계 불명확
  • ❌ 긴 컨텍스트 → 높은 비용
  • ❌ 한 부분 수정 시 전체 재실행

멀티에이전트 방식의 장점

  • ✅ 에이전트별 책임 추적 가능
  • ✅ 전문화 분리 → 각자 고품질
  • ✅ 개인화는 Strategist만 담당
  • ✅ 병렬 실행 → 60초 목표
  • ✅ 에이전트 단위 교체·개선 가능

ADR-003: 6-에이전트 구조 채택 이유 — Supervisor Pattern으로 Strategist가 전체를 종합하되,
각 Specialist는 자신의 영역에서만 판단함으로써 책임을 명확히 분리

APPENDIX A1

AgentState — 에이전트 간 데이터 계약

class AgentState(BaseModel):
    # 초기 입력
    user_query:   str
    user_profile: UserProfile
    portfolio:    Portfolio
    as_of_date:   str | None

    # 에이전트가 순서대로 채움
    curator:    CuratorResult    | None
    user_request: UserRequest    | None
    quant:      QuantResult      | None
    qual:       QualResult       | None
    competitor: CompetitorResult | None
    strategist: StrategistResult | None
    guardrail:  GuardrailResult  | None

설계 원칙

  • 🔒
    Pydantic 강타입으로 에이전트 간 계약 강제
  • 📌
    None 체크로 실행 순서 보장 (Curator 없으면 Quant 실행 불가)
  • 🔄
    LangGraph StateGraph에서 단일 상태 객체로 공유
  • 🧪
    각 에이전트 단독 테스트 시 AgentState Mock 주입

Quant 지표 산식 상세

지표산식소스판단 기준 (MVP)
PER시가총액 ÷ 당기순이익pykrx + DART섹터 평균 대비 저/고평가
PBR시가총액 ÷ 순자산pykrx + DART1.0 미만: 자산 대비 저평가
ROE당기순이익 ÷ 자기자본 × 100DART 재무제표>10%: 수익성 우수
부채비율총부채 ÷ 자기자본 × 100DART 재무제표<150%: 안정, >150%: 주의
매출성장률(당기매출 - 전기매출) ÷ 전기매출 × 100DART 재무제표>5%: 성장세 양호
영업이익률영업이익 ÷ 매출액 × 100DART 재무제표섹터 평균 대비 비교

⏱️ 예상 시간: 각 에이전트 5~10초 × 순차 = 40~60초+

목표: LangGraph StateGraph

from langgraph.graph import StateGraph

graph = StateGraph(AgentState)
graph.add_node("curator",    run_curator)
graph.add_node("classifier", run_classifier)
graph.add_node("quant",     run_quant)
graph.add_node("qual",      run_qual)
graph.add_node("competitor",run_competitor)
graph.add_node("strategist",run_strategist)

# 병렬 Fan-out
graph.add_edge("classifier", [
    "quant", "qual", "competitor"
])
⚡ 예상 시간: 병렬 실행 → 가장 느린 에이전트 시간 ≈ 15~20초
