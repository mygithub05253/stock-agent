## 0. 문서 메타 정보

본 문서는 제품 정식 기능이라기보다 **중간 시연을 위한 검증 모드**를 정의한다. 목적은 AI 주가 예측의 신뢰성을 즉석에서 보여주기 위해, 이미 정답이 확정된 직전 과거일을 타깃 예측일로 삼고 AI에게는 해당 일자의 결과를 숨긴 상태에서 판단을 생성하게 하는 것이다.

---

## 1. 기능 개요

사용자는 Streamlit UI에서 종목과 분석 단위를 선택한다. 시스템은 타깃 예측일을 `2026-05-22`로 고정하고, AI 입력 데이터는 `2026-05-21 23:59` 이전으로 마스킹한다. AI가 BUY/HOLD/SELL 판단과 근거를 출력한 뒤, 화면 하단에서 2026년 5월 22일 실제 주가와 수익률을 대조한다.

| 타깃 예측일 | 2026-05-22 |

| 금지 | 2026-05-22 및 이후 데이터의 AI 컨텍스트 주입 |

## 3. 트리거

- 사용자가 Streamlit 사이드바에서 `백테스팅 검증 모드`를 선택한다.
- 사용자가 종목을 `삼성전자`로 선택한다.
- 사용자가 분석 단위를 `1주일 단위 분석`, `2주일 단위 분석`, `4주일 단위 분석` 중 하나로 선택한다.
- 사용자가 `AI 예측 검증 실행` 버튼을 클릭한다.

| 비용 정책 | OpenRouter 도입 전에는 mock 또는 기존 LLM 추상화 계층으로 대체 가능 |

## 5. 입력

| 필드 | 타입 | 필수 | 기본값 | 설명 |
|------|------|------|--------|------|
| `ticker` | string | Y | `005930` | 분석 종목 코드 |
| `company_name` | string | Y | `삼성전자` | 화면 표시용 종목명 |
| `target_date` | date | Y | `2026-05-22` | AI가 예측해야 하는 날짜 |
| `mask_datetime` | datetime | Y | `2026-05-21 23:59:59` | AI 입력 데이터 상한 |
| `analysis_window` | enum | Y | `1w` | `1w`, `2w`, `4w` |
| `lookback_days` | int | Y | `7` | 분석 단위별 입력 기간 |
| `data_sources` | list | Y | price, news, dart, macro | 사용할 데이터 범주 |
| `force_refresh` | boolean | N | false | 캐시 무시 여부 |

## 6. 처리 흐름

1. **설정 생성**
   - UI 선택값으로 `BacktestConfig`를 생성한다.
   - `analysis_window`에 따라 `lookback_days`를 7, 14, 28로 매핑한다.
   - `input_start_date = target_date - lookback_days`를 계산한다.
   - `input_end_datetime = target_date - 1일 23:59:59`로 고정한다.

2. **데이터 슬라이싱**
   - 가격 데이터는 `base_date >= input_start_date` AND `base_date <= 2026-05-21` 조건으로 조회한다.
   - 뉴스/DART/매크로 데이터는 `published_at <= mask_datetime` 조건으로 조회한다.
   - `target_date >= 2026-05-22` 데이터는 AI 입력에서 제외한다.
   - 제외된 레코드 수를 `leakage_guard.excluded_count`로 기록한다.

3. **프롬프트 구성**
   - AI의 현재 시점을 `2026-05-21 23:59`로 고정한다.
   - 제공된 데이터 기간을 명시한다.
   - 2026년 5월 22일 결과를 모른다는 조건을 명시한다.
   - 출력 형식은 판단, 신뢰도, 핵심 근거, 리스크, 데이터 한계로 제한한다.

4. **에이전트 실행**
   - Quant Worker는 입력 기간 주가 흐름, 거래량, 변동성, 모멘텀을 요약한다.
   - Qual Worker는 입력 기간 뉴스/공시 RAG 근거를 요약한다.
   - Strategist는 BUY/HOLD/SELL 판단과 신뢰도를 생성한다.
   - Guardrail은 미래 데이터 언급, 단정적 투자권유 표현, 출처 누락을 검증한다.

5. **실제 결과 대조**
   - AI 판단 생성 이후 별도 단계에서 2026-05-22 실제 OHLCV를 조회한다.
   - 전일 종가 대비 타깃일 수익률을 계산한다.
   - AI 방향과 실제 방향을 비교해 `hit`, `miss`, `neutral` 중 하나로 표시한다.

6. **화면 렌더링**
   - 상단에는 선택 조건과 마스킹 기준을 표시한다.
   - 중단에는 AI 판단 카드와 RAG 근거 보고서를 표시한다.
   - 하단에는 실제 결과 차트, 수익률, 예측 적중 여부를 표시한다.
   - 데이터 누수 방지 배지를 함께 표시한다.

## 7. 출력

| 출력 영역 | 필드 | 설명 |
|-----------|------|------|
| 설정 배지 | `target_date`, `mask_datetime`, `analysis_window` | 시연자가 마스킹 조건 설명 |
| AI 판단 | `signal`, `confidence`, `rationale`, `risks` | AI가 5월 22일을 예측한 결과 |
| RAG 보고서 | `news_evidence`, `dart_evidence`, `macro_context` | 입력 기간 근거 |
| 누수 검증 | `included_count`, `excluded_count`, `latest_input_timestamp` | 미래 데이터 배제 증거 |
| 실제 결과 | `actual_open`, `actual_close`, `return_pct`, `direction` | 5월 22일 정답 |
| 검증 결과 | `prediction_hit`, `comparison_summary` | AI 예측과 실제 결과 대조 |

## 8. 예외 처리

| 상황 | 처리 |
|------|------|
| 입력 기간 데이터 부족 | 데이터 부족 경고 표시, 판단 신뢰도 하향 |
| 5월 22일 실제 결과 없음 | AI 판단까지만 표시하고 정답 대조 영역은 비활성화 |
| 뉴스/DART RAG 없음 | 가격/매크로 기반 제한 분석으로 진행 |
| 슬라이서가 5월 22일 이후 데이터를 발견 | AI 실행 중단, 누수 위험 경고 |
| LLM 호출 실패 | mock Strategist 또는 규칙 기반 판단으로 대체 |
| OpenRouter 미설정 | 기존 LLM 추상화 또는 mock 응답 사용 |

## 10. 개발 담당 순서

1. 데이터팀: 최근 6개월 가격/DART/뉴스/매크로 수집 및 Supabase 적재.
2. 백엔드: `BacktestConfig`, 데이터 슬라이서, 실제 결과 대조 로직 구현.
3. 에이전트: Quant/Qual/Strategist/Guardrail skeleton 연결.
4. 프론트엔드: Streamlit UI와 데모 카드 구현.
5. PM: 시연 스토리보드, 문서, HTML 아키텍처, README 업데이트.
