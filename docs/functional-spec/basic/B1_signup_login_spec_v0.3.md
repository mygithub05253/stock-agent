## 0. 문서 메타 정보

본 문서는 PRD 기준 기본 기능 `B1. 회원가입/로그인`의 상세 기능 명세서다. 기존 `overview/functional_spec_all_features_v0.1.md`의 7대 표준 양식인 **트리거 / 전제조건 / 입력 / 처리 흐름 / 출력 / 예외 처리 / 담당**을 유지하고, `basic/B5_portfolio_bulk_advice_spec_v0.2.md`에서 확립한 비용 상한 및 예외 처리 기준을 동일하게 적용한다.

`B1`은 LLM 호출이 필요 없는 인증/프로필 등록 기능이다. 따라서 월 LLM 비용 5만원 상한에 직접 비용을 추가하지 않도록 **LLM 호출 0회**를 원칙으로 하며, 개인정보 보호와 로그 마스킹은 Guardrail Agent 또는 규칙 기반 필터로 처리한다.

## 2. 기능 개요

회원가입/로그인은 사용자가 stock-agent의 개인화 분석 기능을 사용하기 위한 진입 기능이다. 신규 사용자는 이메일과 비밀번호로 계정을 만들고, 투자성향과 관심 섹터를 등록한다. 기존 사용자는 이메일과 비밀번호로 로그인하여 세션을 발급받고, 이후 보유 종목 등록, 포트폴리오 일괄 안내, 종목 추천, BUY/HOLD/SELL 안내에서 사용자 프로필을 재사용한다.

MVP에서는 SNS/카카오/네이버 로그인은 제외하고 자체 이메일+비밀번호 인증만 지원한다. 사용자의 투자성향과 관심 섹터는 추천 및 적합도 판단의 기본 컨텍스트로 쓰이며, 모든 개인정보성 로그는 마스킹 후 저장한다.

## 4. B1. 회원가입/로그인

### 4.1 트리거

- 신규 사용자가 홈 화면에서 `회원가입` 버튼을 클릭한다.
- 기존 사용자가 홈 화면에서 `로그인` 버튼을 클릭한다.
- 세션이 만료된 사용자가 보호된 메뉴인 `보유 종목 관리`, `포트폴리오 일괄 안내`, `종목 추천` 등에 접근한다.
- 시연 계정 사용자가 데모 로그인 버튼 또는 사전 등록된 테스트 계정으로 로그인한다.

### 4.2 전제조건

| 조건 | 상세 |
|------|------|
| DB 준비 | Postgres에 `users` 테이블이 존재해야 한다. |
| 이메일 중복 체크 | `users.email`에 unique 제약 또는 중복 조회 로직이 있어야 한다. |
| 비밀번호 해싱 | 비밀번호는 평문 저장 금지. bcrypt 또는 동등 수준의 해싱 방식을 사용한다. |
| 세션 저장소 | Streamlit `st.session_state` 또는 향후 서버 세션에 `user_id`, 로그인 상태, 프로필 요약을 저장할 수 있어야 한다. |
| 관심 섹터 기준 | PRD 기준 MVP 관심 섹터인 반도체/금융/소비재를 우선 지원한다. |
| PII 로그 정책 | 이메일, 이름, 생년월일, 비밀번호는 원문 로그 저장 금지. 로그 저장 시 마스킹한다. |
| 비용 정책 | B1은 LLM 호출 없이 동작해야 하며 `llm_cost_log`에 비용을 발생시키지 않는다. |

### 4.3 입력

회원가입 입력:

| 필드 | 타입 | 필수 | 검증 기준 | 설명 |
|------|------|------|-----------|------|
| `email` | string | 필수 | 이메일 형식, 중복 불가, 소문자 정규화 | 로그인 ID로 사용 |
| `password` | string | 필수 | 8자 이상, 공백만 입력 불가 | 저장 전 해싱 필수 |
| `password_confirm` | string | 필수 | `password`와 일치 | 회원가입 화면에서만 사용, DB 저장 금지 |
| `name` | string | 필수 | 1~50자, 제어문자 불가 | 화면 인사말 및 프로필 표시 |
| `birth_date` | date | 선택 | YYYY-MM-DD, 미래 날짜 불가 | 연령대 기반 적합도 확장 시 사용 |
| `risk_profile` | enum | 필수 | `보수`, `중립`, `공격` 중 하나 | 투자성향 |
| `investment_horizon` | enum | 필수 | `단기`, `중기`, `장기` 중 하나 | 투자 기간 |
| `target_return` | float | 선택 | 0.00~0.50 권장 | 목표 수익률 |
| `max_drawdown_tolerance` | float | 선택 | -0.50~0.00 권장 | 허용 손실률 |
| `interest_sectors` | list[string] | 필수 | 1개 이상, MVP 섹터 우선 | 관심 섹터 목록 |
| `terms_agreed` | boolean | 필수 | `true` | 서비스 이용/면책 고지 동의 |
| `privacy_agreed` | boolean | 필수 | `true` | 개인정보 처리 동의 |

로그인 입력:

| 필드 | 타입 | 필수 | 검증 기준 | 설명 |
|------|------|------|-----------|------|
| `email` | string | 필수 | 이메일 형식, 소문자 정규화 | 가입 계정 조회 |
| `password` | string | 필수 | 빈 값 불가 | 해시 검증에 사용 |
| `remember_me` | boolean | 선택 | 기본값 `false` | MVP에서는 세션 유지 시간만 조정 |

시스템 생성 값:

| 필드 | 타입 | 생성 시점 | 설명 |
|------|------|-----------|------|
| `user_id` | uuid | 회원가입 성공 시 | 사용자 고유 식별자 |
| `password_hash` | string | 회원가입 성공 시 | bcrypt 해시 값 |
| `created_at` | datetime | 회원가입 성공 시 | 가입 시각 |
| `updated_at` | datetime | 프로필 변경 시 | 최근 수정 시각 |
| `last_login_at` | datetime | 로그인 성공 시 | 최근 로그인 시각 |
| `failed_login_count` | int | 로그인 실패 시 | 계정 잠금 판단에 사용 |
| `status` | enum | 회원가입 성공 시 | `active`, `locked`, `deleted` 등 |

### 4.4 처리 흐름

회원가입 처리:

1. **회원가입 폼 표시**
   - 홈 화면 또는 사이드바에서 회원가입 폼을 표시한다.
   - 필수 입력값과 약관/개인정보 동의 체크박스를 함께 제공한다.
   - 투자성향과 관심 섹터는 이후 추천/권유 기능의 컨텍스트로 사용된다는 안내를 표시한다.

2. **입력값 1차 검증**
   - 이메일 형식, 비밀번호 길이, 비밀번호 확인 일치 여부를 검증한다.
   - `risk_profile`, `investment_horizon`, `interest_sectors`가 비어 있으면 제출을 막는다.
   - `target_return`, `max_drawdown_tolerance`는 허용 범위를 벗어나면 경고 후 재입력을 요구한다.

3. **PII/보안 필드 처리**
   - 비밀번호와 비밀번호 확인 값은 로그, 세션, DB 평문 저장 대상에서 제외한다.
   - 이름, 이메일, 생년월일은 로그 저장 전 마스킹한다.
   - Guardrail Agent를 호출하지 않고 규칙 기반 마스킹을 우선 적용한다.

4. **이메일 중복 확인**
   - `users` 테이블에서 정규화된 이메일로 기존 계정을 조회한다.
   - 동일 이메일이 존재하면 회원가입을 중단하고 로그인 유도 메시지를 표시한다.

5. **비밀번호 해싱**
   - 비밀번호는 bcrypt 또는 동등 수준의 해시 알고리즘으로 변환한다.
   - 해시에는 salt를 적용하며, 평문 비밀번호는 즉시 폐기한다.

6. **사용자 프로필 저장**
   - `users` 테이블에 `email`, `password_hash`, `name`, `birth_date`, `risk_profile`, `investment_horizon`, `target_return`, `max_drawdown_tolerance`, `interest_sectors`, `created_at`, `status`를 저장한다.
   - `interest_sectors`는 배열/JSON 또는 별도 매핑 테이블로 저장할 수 있으나 MVP에서는 `users.interest_sectors` JSON/array 형태를 우선한다.

7. **세션 발급**
   - 회원가입 성공 직후 자동 로그인 처리한다.
   - `st.session_state`에 `user_id`, `email_masked`, `name`, `risk_profile`, `interest_sectors`, `is_authenticated=true`를 저장한다.

8. **완료 화면 렌더링**
   - 회원가입 완료 메시지와 다음 행동인 `보유 종목 등록하기` 또는 `종목 추천 보기` 버튼을 제공한다.
   - 로그에는 `signup_success` 이벤트만 남기고 원문 PII는 저장하지 않는다.

로그인 처리:

1. **로그인 폼 표시**
   - 이메일과 비밀번호 입력 필드를 표시한다.
   - 회원가입하지 않은 사용자를 위해 회원가입 이동 링크를 제공한다.

2. **입력값 검증**
   - 이메일 형식과 비밀번호 빈 값 여부를 확인한다.
   - 형식 오류가 있으면 DB 조회 없이 즉시 사용자에게 안내한다.

3. **계정 조회**
   - 정규화된 이메일로 `users` 테이블을 조회한다.
   - 계정이 없더라도 보안을 위해 `이메일 또는 비밀번호가 올바르지 않습니다`처럼 일반화된 메시지를 사용한다.

4. **계정 상태 확인**
   - `status=locked`이면 로그인 검증을 중단하고 잠금 안내를 표시한다.
   - `status=deleted` 또는 비활성 계정이면 로그인 불가 메시지를 표시한다.

5. **비밀번호 해시 검증**
   - 입력 비밀번호와 저장된 `password_hash`를 비교한다.
   - 실패 시 `failed_login_count`를 증가시키고, 일정 횟수 이상 실패하면 임시 잠금 상태로 전환한다.

6. **로그인 성공 처리**
   - `last_login_at`, `failed_login_count=0`을 업데이트한다.
   - `st.session_state`에 사용자 세션 정보를 저장한다.
   - 사용자가 진입하려던 보호 메뉴가 있으면 해당 화면으로 이동하고, 없으면 홈 또는 포트폴리오 요약 화면으로 이동한다.

7. **로그아웃 처리**
   - 사용자가 `로그아웃` 버튼을 클릭하면 `st.session_state`의 인증 관련 값을 삭제한다.
   - 로그아웃 후 홈 화면으로 이동한다.

비용/LLM 처리:

1. B1은 회원 인증 기능이므로 LLM을 호출하지 않는다.
2. PII 마스킹, 입력 검증, 위험 문구 처리는 정규식/서버 로직으로 처리한다.
3. `llm_cost_log`에는 B1 관련 비용 로그를 남기지 않는다.
4. 향후 온보딩 문구 개인화가 추가되더라도 기본 로그인 경로에서는 LLM 호출을 금지한다.

### 4.6 예외 처리

| 예외 상황 | 감지 조건 | 처리 방안 | 사용자 메시지 |
|-----------|-----------|-----------|---------------|
| 이메일 형식 오류 | 이메일 정규식 검증 실패 | DB 조회 없이 폼 유지 | `이메일 형식을 확인해 주세요.` |
| 이메일 중복 | `users.email` 기존 행 존재 | 회원가입 중단, 로그인 유도 | `이미 가입된 이메일입니다. 로그인해 주세요.` |
| 비밀번호 8자 미만 | `len(password) < 8` | 폼 유지, 비밀번호 재입력 요구 | `비밀번호는 8자 이상으로 입력해 주세요.` |
| 비밀번호 확인 불일치 | `password != password_confirm` | 폼 유지, 확인 필드 재입력 요구 | `비밀번호 확인 값이 일치하지 않습니다.` |
| 필수 프로필 누락 | 투자성향/기간/관심섹터 없음 | 제출 차단 | `투자성향과 관심 섹터를 선택해 주세요.` |
| 관심 섹터 0개 | `interest_sectors` 빈 배열 | 제출 차단 | `관심 섹터를 1개 이상 선택해 주세요.` |
| 목표수익률 범위 초과 | `target_return > 0.50` 등 | 경고 후 재입력 또는 기본값 처리 | `목표수익률 범위를 다시 확인해 주세요.` |
| 허용 손실률 범위 초과 | `max_drawdown_tolerance < -0.50` 등 | 경고 후 재입력 또는 기본값 처리 | `허용 손실률 범위를 다시 확인해 주세요.` |
| 개인정보 동의 누락 | `privacy_agreed=false` | 회원가입 차단 | `개인정보 처리 동의가 필요합니다.` |
| 약관 동의 누락 | `terms_agreed=false` | 회원가입 차단 | `서비스 이용 및 면책 고지에 동의해 주세요.` |
| 존재하지 않는 계정 | 이메일 조회 결과 없음 | 일반화된 로그인 실패 메시지 | `이메일 또는 비밀번호가 올바르지 않습니다.` |
| 비밀번호 불일치 | 해시 검증 실패 | 실패 횟수 증가, 일반화 메시지 | `이메일 또는 비밀번호가 올바르지 않습니다.` |
| 반복 로그인 실패 | 실패 횟수 임계값 초과 | 계정 임시 잠금 또는 지연 응답 | `로그인 시도가 여러 번 실패했습니다. 잠시 후 다시 시도해 주세요.` |
| 잠긴 계정 | `status=locked` | 로그인 차단 | `보안상 계정이 잠시 잠겼습니다.` |
| 삭제/비활성 계정 | `status=deleted` 등 | 로그인 차단 | `사용할 수 없는 계정입니다.` |
| DB 연결 실패 | DB connection error | 회원가입/로그인 중단, 에러 로그 저장 | `일시적으로 로그인할 수 없습니다. 잠시 후 다시 시도해 주세요.` |
| 세션 만료 | 보호 메뉴 접근 시 인증 정보 없음 | 로그인 화면으로 리다이렉트 | `로그인이 필요합니다.` |
| PII 로그 노출 위험 | 로그에 원문 이메일/이름/생년월일 포함 감지 | 저장 전 마스킹, 원문 로그 차단 | 사용자 노출 없음 |
| LLM 호출 발생 위험 | B1 처리 중 LLM 라우터 호출 시도 | 호출 차단, 개발 로그에 정책 위반 기록 | 사용자 노출 없음 |
