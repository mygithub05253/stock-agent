## 0. 문서 메타 정보

본 문서는 PRD 기준 기본 기능 `B4. 종목 기본 정보 조회`의 상세 기능 명세서다. 기존 `overview/functional_spec_all_features_v0.1.md`의 7대 표준 양식인 **트리거 / 전제조건 / 입력 / 처리 흐름 / 출력 / 예외 처리 / 담당**을 유지하고, B3 검색 기능 및 B5 포트폴리오 일괄 안내 기능과의 연결을 명확히 정의한다.

`B4`는 사용자가 본격적인 고급 분석을 실행하기 전에 종목의 기본 상태를 빠르게 파악하는 화면이다. 따라서 복잡한 LLM 분석을 수행하지 않고, `company`, `stock_price`, `financial_data`/`financial_statement`, `rag_documents`/`rag_chunks`에 이미 적재된 데이터를 기반으로 차트, 핵심 재무지표, 최근 뉴스 헤드라인을 보여준다.

## 2. 기능 개요

종목 기본 정보 조회 기능은 사용자가 B3 종목 검색 결과나 B2 보유 종목 표에서 특정 종목을 선택했을 때, 해당 종목의 기본 정보를 한 화면에서 보여주는 기능이다.

MVP 화면은 기업 기본 정보, 1년 주가 차트, 핵심 재무지표 카드, 최근 가격/거래 정보, 최근 7일 뉴스 헤드라인, 보유 여부, 후속 분석 CTA로 구성한다. 이 화면은 사용자가 “이 종목이 어떤 회사인지”를 빠르게 파악하고, 필요 시 A1 5개년 밸류에이션, A2 산업·정성 분석, A4 BUY/HOLD/SELL 권유로 이동하는 관문 역할을 한다.

## 4. B4. 종목 기본 정보 조회

### 4.1 트리거

- 사용자가 B3 종목 검색 결과에서 특정 종목을 클릭한다.
- 사용자가 B2 보유 종목 표에서 종목명을 클릭한다.
- 사용자가 직접 URL 또는 내부 상태로 `stock_code`가 포함된 종목 상세 화면에 진입한다.
- 사용자가 B5 포트폴리오 일괄 안내 결과에서 특정 종목의 `기본 정보 보기`를 클릭한다.

### 4.2 전제조건

| 조건 | 상세 |
|------|------|
| 종목 코드 확정 | `stock_code`가 6자리이고 B3 검색 또는 내부 링크를 통해 확정되어야 한다. |
| 종목 마스터 | `company` 테이블에 해당 `stock_code`가 존재해야 한다. |
| 시세 데이터 | 1년 주가 차트를 표시하려면 `stock_price`에 기간별 일봉 데이터가 있어야 한다. |
| 재무 데이터 | PER/PBR/ROE 등 지표를 표시하려면 `financial_data` 또는 `financial_statement`가 있어야 한다. |
| 뉴스/RAG 데이터 | 최근 뉴스 헤드라인을 표시하려면 `rag_documents` 또는 `rag_chunks`에 뉴스 메타가 있어야 한다. |
| 보유 여부 | 로그인 사용자의 보유 상태를 표시하려면 `holdings` 테이블 조회가 가능해야 한다. |
| 비용 정책 | B4는 신규 LLM 호출 없이 동작해야 하며 `llm_cost_log`에 비용을 발생시키지 않는다. |

### 4.3 입력

| 필드 | 타입 | 필수 | 출처 | 설명 |
|------|------|------|------|------|
| `stock_code` | string(6) | 필수 | B3 검색 결과, B2 보유 종목, URL state | 조회 대상 종목 코드 |
| `user_id` | uuid/string | 선택 | 세션 | 보유 여부/평균 매수가 표시 시 사용 |
| `price_period` | enum | 선택 | UI | `1M`, `3M`, `6M`, `1Y` 중 하나. MVP 기본값 `1Y` |
| `include_news` | boolean | 선택 | UI/시스템 | 최근 뉴스 헤드라인 포함 여부. 기본값 `true` |
| `include_holding` | boolean | 선택 | UI/시스템 | 사용자 보유 정보 포함 여부. 기본값 로그인 시 `true` |
| `news_days` | int | 선택 | 시스템 설정 | 최근 뉴스 조회 기간. 기본값 7일 |
| `news_limit` | int | 선택 | 시스템 설정 | 표시할 뉴스 수. 기본값 5 |

조회 결과 내부 필드:

| 필드 | 타입 | 출처 | 설명 |
|------|------|------|------|
| `corp_name` | string | `company` | 종목명 |
| `corp_code` | string/null | `company` | DART 고유번호 |
| `sector` | string/null | `company` | 섹터 |
| `current_price` | decimal/null | `stock_price` | 최근 가격 |
| `price_series` | list | `stock_price` | 차트용 일봉 데이터 |
| `market_cap` | decimal/null | `stock_price` | 시가총액 |
| `per` | decimal/null | 계산 | PER |
| `pbr` | decimal/null | 계산 | PBR |
| `roe` | decimal/null | 계산 | ROE |
| `dividend_yield` | decimal/null | 계산/데이터 | 배당수익률 |
| `news_headlines` | list | `rag_documents`/`rag_chunks` | 최근 뉴스 헤드라인 |
| `holding_summary` | object/null | `holdings` | 사용자 보유 수량, 평균 매수가, 평가손익 |

### 4.4 처리 흐름

1. **화면 진입 및 입력 확인**
   - UI는 `stock_code`를 수신한다.
   - `stock_code`가 없거나 6자리 형식이 아니면 B3 검색 화면으로 되돌린다.
   - 세션에 `user_id`가 있으면 보유 여부 조회 플래그를 활성화한다.

2. **종목 마스터 조회**
   - `company` 테이블에서 `stock_code` 기준으로 `corp_name`, `corp_code`, `sector`를 조회한다.
   - 조회 결과가 없으면 종목 없음 상태를 표시하고 B3 검색으로 이동할 수 있게 한다.

3. **시세 데이터 조회**
   - `stock_price`에서 `price_period`에 해당하는 일봉 데이터를 조회한다.
   - MVP 기본 기간은 1년이다.
   - 가장 최근 거래일의 `close_price`, `volume`, `market_cap`을 조회한다.
   - 전일 데이터가 있으면 등락률을 계산한다.

4. **주가 차트 데이터 생성**
   - 조회된 일봉 데이터를 날짜순으로 정렬한다.
   - 차트용 데이터 구조인 `date`, `close_price`, `volume` 배열을 생성한다.
   - 데이터가 일부 누락되어도 가능한 기간만 표시하고, 차트에 데이터 부족 상태를 함께 표시한다.

5. **핵심 재무 데이터 조회**
   - `financial_data` 또는 `financial_statement`에서 최근 연도/분기 기준 매출, 영업이익, 순이익, 자본, EPS/BPS 등 계산 가능한 항목을 조회한다.
   - 데이터 소스가 `financial_statement`와 `financial_data`로 나뉘어 있을 수 있으므로, 실제 구현 시 현재 DB 스키마에 맞춰 우선순위를 정한다.
   - 재무 데이터가 없으면 재무지표 카드는 `공시 전` 또는 `데이터 없음`으로 표시한다.

6. **핵심 지표 계산**
   - PER, PBR, ROE, 배당수익률 등은 LLM이 아니라 Python/SQL로 계산한다.
   - 필요한 분모/분자 데이터가 없으면 해당 지표만 `N/A`로 표시한다.
   - 지표 계산식과 기준일은 개발 문서 또는 tooltip에서 확인 가능하게 한다.

7. **최근 뉴스/공시 헤드라인 조회**
   - `include_news=true`이면 최근 `news_days`일 기준 뉴스/공시 메타를 조회한다.
   - 우선 `rag_documents`에서 문서 단위 메타와 제목을 조회하고, 필요 시 `rag_chunks`에서 headline metadata를 조회한다.
   - B4에서는 LLM 요약을 새로 생성하지 않고 제목, 출처, 날짜, URL만 표시한다.
   - 뉴스가 없으면 `최근 뉴스 없음` 상태를 표시한다.

8. **사용자 보유 정보 조회**
   - 로그인 사용자이고 `include_holding=true`이면 `holdings`에서 `user_id + stock_code`를 조회한다.
   - 보유 중이면 평균 매수가, 수량, 평가손익, 수익률을 계산해 표시한다.
   - 보유하지 않은 종목이면 `보유 종목에 추가` 버튼을 표시한다.

9. **후속 CTA 구성**
   - `보유 종목에 추가` 버튼은 B2 등록 폼으로 연결한다.
   - `이 종목 분석하기` 버튼은 A1/A2/A4 분석 플로우 진입점으로 연결한다.
   - `포트폴리오 일괄 안내에서 보기` 버튼은 보유 중인 경우에만 표시한다.

10. **Guardrail/면책 문구 적용**
    - B4는 정보 조회 화면이므로 투자 권유 표현을 생성하지 않는다.
    - 화면 하단에 `본 정보는 투자 판단 참고용이며 매수/매도 권유가 아닙니다` 문구를 표시한다.

11. **비용/LLM 처리**
    - B4는 기본 정보 조회 기능이므로 신규 LLM을 호출하지 않는다.
    - 뉴스 헤드라인은 캐시/DB 조회만 사용한다.
    - 정성 해석, 호재/악재 분류, 사업보고서 요약은 A2 기능으로 분리한다.

| 종목 코드 없음 | `stock_code` null/empty | B3 검색 화면으로 이동 | `조회할 종목을 먼저 선택해 주세요.` |

| 종목 마스터 없음 | `company` 조회 결과 0건 | 종목 없음 상태 표시 | `지원하지 않는 종목입니다. 다시 검색해 주세요.` |
| 시세 데이터 없음 | `stock_price` 조회 결과 0건 | 차트/가격 카드 대신 데이터 없음 표시 | `시세 데이터가 아직 준비되지 않았습니다.` |
| 1년 차트 일부 결손 | 기간 내 일부 거래일 누락 | 가능한 데이터만 차트 표시, 경고 배지 표시 | `일부 기간의 시세 데이터가 누락되었습니다.` |
| 재무 데이터 없음 | `financial_data`/`financial_statement` 조회 결과 없음 | 재무 카드 `공시 전` 또는 `데이터 없음` 표시 | `재무 데이터가 아직 준비되지 않았습니다.` |
| 특정 재무지표 계산 불가 | EPS/BPS/자본 등 필요 값 없음 | 해당 지표만 `N/A` 표시 | `일부 지표는 계산에 필요한 데이터가 부족합니다.` |
| 뉴스 0건 | 최근 `news_days` 내 문서 없음 | 뉴스 영역 빈 상태 표시 | `최근 뉴스가 없습니다.` |
| RAG 테이블 미적재 | `rag_documents`/`rag_chunks` 없음 | 뉴스 영역 숨김 또는 준비 중 표시 | `뉴스 데이터가 준비되지 않았습니다.` |
| 보유 정보 없음 | 로그인 사용자이나 `holdings` 조회 결과 없음 | 보유 정보 대신 추가 버튼 표시 | `아직 보유 종목에 등록되지 않았습니다.` |
| DB 조회 실패 | DB connection/query error | 화면 부분 실패 처리, 에러 로그 저장 | `일시적으로 종목 정보를 불러올 수 없습니다.` |
| LLM 호출 발생 위험 | B4 처리 중 LLM 라우터 호출 시도 | 호출 차단, 개발 로그에 정책 위반 기록 | 사용자 노출 없음 |

## 6. KPI / 비용 상한 / 운영 기준

| 구분 | 지표 | 목표/임계값 | 비고 |
|------|------|-------------|------|
| 비용 | B4 신규 LLM 호출 수 | 0회 | 월 5만원 상한 보호 |
| 성능 | 기본 정보 응답 시간 | 2초 이내 목표 | DB 정상 기준 |
| 성능 | 차트 렌더링 시간 | 3초 이내 목표 | 1년 일봉 기준 |
| 데이터 품질 | 종목 마스터 조회 성공률 | 99% 이상 | B3 검색 결과 기준 |
| 데이터 품질 | 시세 데이터 표시율 | 95% 이상 목표 | 수집 상태 의존 |
| 데이터 품질 | 재무지표 표시율 | 80% 이상 목표 | 신규상장/미제출 제외 |
| UX | 부분 데이터 결손 표시 | 100% | 숨기지 않고 상태 표시 |
| 연계성 | 후속 분석 CTA 노출 | 100% | A1/A2/A4 진입 |

운영 정책:

- B4는 고급 분석 전의 빠른 확인 화면이므로 LLM 분석을 수행하지 않는다.
- 시세, 재무, 뉴스 중 일부가 없어도 전체 화면을 실패시키지 않고 부분 렌더링한다.
- 투자 판단을 유도하는 BUY/HOLD/SELL 문구는 B4에서 표시하지 않는다.
- 뉴스 헤드라인은 출처와 날짜를 함께 표시한다.
- B4에서 사용자가 상세 분석을 원하면 A1, A2, A4로 명확히 이동시킨다.
- 데이터 기준일을 화면에 표시하여 시세/재무 정보의 신뢰 범위를 알 수 있게 한다.
