## 0. 이 문서를 보는 법 (비전공자용)

> **ADR (Architecture Decision Record)** = "왜 이 선택을 했는가" 를 기록하는 문서.
> 본 ADR은 *결정 전 상태* 입니다. 양쪽 안의 장단점을 모두 자세히 적고, 팀 회의에서 한쪽 채택 후 본 문서를 **Accepted** 로 변경합니다.

**한 줄 요약:** AI가 자기 답을 *비판적으로 재검토* 하는 단계를 별도 에이전트(Critic)로 분리할지, Strategist 안의 한 단계로 합칠지의 의사결정.

---

## 1. 배경 (Context)

### 1.1 무엇을 결정해야 하나?

우리 시스템은 사용자에게 BUY/HOLD/SELL 권유를 합니다. 권유의 신뢰도를 높이려면 *한 번 결정한 다음, 그 결정을 다른 시각으로 다시 검증하는 단계* 가 필요합니다. 이를 **Adversarial Critic** (적대적 검증) 이라고 부릅니다.

문제는 이 검증 단계를 **별도 에이전트로 분리하느냐, Strategist 안에 한 단계로 넣느냐** 입니다.

### 1.2 왜 검증이 필요한가? (교수 피드백 #4)

> 교수님: "투자에 대해서 벨류에이션하고 권유하는 구조는 좋지만 그렇다면 그거에 대한 근거 설명을 상세하게 해주는 것이 굉장히 중요합니다."

근거 설명의 *신뢰도* 를 높이려면 다음 중 하나가 필요:
- **외부 비판자 시뮬레이션**: "회계 보수주의자라면 이 가정에 어떻게 반박할까?"
- **자체 검토**: "내가 놓친 리스크가 있나?"

전자가 Critic Agent 패턴, 후자는 Strategist 안의 한 단계.

## 2. 옵션 A — Critic Agent 분리 (5종 페르소나)

### 2.1 구조

```
사용자 입력 → Curator → [Quant + Qual + Competitor 병렬]
            → Strategist (1차 결정)
            → 🆕 Critic Agent (5 페르소나로 동시 공격)
                ├─ 회계 보수주의자
                ├─ 매크로 비관론자
                ├─ 경쟁구도 회의론자
                ├─ 규제·ESG 평가관
                └─ 모멘텀 트레이더
            → Strategist (재합성: 공격 반영)
            → Guardrail
```

### 2.2 동작 예시 (박민호 페르소나 시연)

1. **Strategist 1차 결정**: "삼성전자 BUY 78% / 적정가 80,000원"
2. **Critic 5인 동시 공격**:
   - 회계 보수주의자: "5y 가정 OPM 0.5%p 개선이 너무 낙관적. CAPEX 확대 부담 미반영"
   - 매크로 비관론자: "환율 1,400원 시나리오 누락. 적정가 -15% 가능"
   - 경쟁구도 회의론자: "TSMC 점유율 추가 잠식 시 글로벌 시장 위축"
   - 규제·ESG 평가관: "미·중 반도체 규제 강화 시 CHIPS Act 영향 미반영"
   - 모멘텀 트레이더: "단기 RSI 과열, 6개월 누적 +25% — 기술적 조정 가능"
3. **Strategist 재합성**: 5 공격 중 합리적인 것 반영 → "BUY 70% (78%에서 하향) / 적정가 76,000원 (80,000원에서 하향)"

### 2.3 장점 (Pros)

| # | 장점 | 설명 |
|---|------|------|
| ✅ | **발표 임팩트 ↑** | "5종 페르소나 적대적 검증" 한 줄이 발표·면접에서 강력. 학습 다양성 강조 |
| ✅ | **다각적 검증** | 단일 LLM이 놓치는 시각 5가지 보완 (다양성 효과) |
| ✅ | **Pydantic 명확화** | Critic 출력이 별도 스키마(`CriticReport`)로 강제 → 디버깅·테스트 용이 |
| ✅ | **A2A 패턴 사례** | 10주차 강사 요구 "A2A 적용" 의 가장 명확한 사례 (Critic ↔ Strategist 협상) |
| ✅ | **모듈성** | Critic을 끄고/켜기 쉬움. Phase 1엔 OFF, Phase 2엔 ON 가능 |
| ✅ | **부트캠프 W3 ReAct 깊이 활용** | Critic도 ReAct로 동작 시 Thought-Action-Observation 학습 효과 ↑ |

### 2.4 단점 (Cons)

| # | 단점 | 설명 |
|---|------|------|
| ❌ | **LLM 호출 5배** | 1회 분석에 Strategist 1회 → Critic 5회 + 재합성 1회 = +6회 (총 13→19회) |
| ❌ | **비용 ~30% 증가** | 단일 분석 36원 → 47원. 12주 5만원 상한에 부담 |
| ❌ | **응답 시간 ↑** | 병렬 호출 가능하지만 동기화 필요 → 5~10초 추가 |
| ❌ | **구현 복잡도 ↑** | 5 페르소나 × 시스템 프롬프트 = 5 파일 관리 |
| ❌ | **품질 보장 어려움** | 각 페르소나가 *"정말 그 시각으로 비판"* 하도록 프롬프트 설계 어려움 (역할 일탈 위험) |
| ❌ | **Phase 1 종료 어려움** | 8주차 MVP에 포함 시 일정 압박. Phase 2(10주차)에 도입해도 충분 |

### 2.5 비용 추산 (분리안 기준)

| 항목 | 통합 안 | 분리 안 (5 페르소나) | 차이 |
|------|---------|------------------------|------|
| 1회 호출 수 | 13회 | 19회 | +46% |
| 1회 비용 (gpt-4o-mini) | 36원 | 53원 | +47% |
| 12주 1,000회 비용 | 36,000원 | 53,000원 | +17,000원 |
| **5만원 상한 안전성** | ✅ 안전 | ⚠️ 거의 한계 | |

### 2.6 구현 예시 (분리 안)

```python
# src/stock_agent/agents/critic.py
class CriticAgent:
    PERSONAS = [
        "accounting_conservative",
        "macro_pessimist",
        "competition_skeptic",
        "regulatory_esg",
        "momentum_trader",
    ]

    async def critique(self, strategist_output) -> CriticReport:
        # 5 페르소나 병렬 호출
        attacks = await asyncio.gather(*[
            self._critique_with_persona(strategist_output, p)
            for p in self.PERSONAS
        ])
        return CriticReport(attacks=attacks)
```

## 3. 옵션 B — Strategist 안에 통합 (단일 검증 단계)

### 3.1 구조

```
사용자 입력 → Curator → [Quant + Qual + Competitor 병렬]
            → Strategist
                ├─ 1단계: 4 입력 종합 (1차 결정)
                ├─ 2단계: 자체 검토 (스스로 5 시각 점검)
                └─ 3단계: 최종 결정
            → Guardrail
```

### 3.2 동작 예시 (박민호 페르소나)

1. **Strategist 1차 결정 (Self-thought)**: "삼성전자 BUY 78% / 적정가 80,000원"
2. **Strategist 2단계 자체 검토 (Self-critique)**: 같은 LLM 호출 안에서 *"잠시, 이 결정에 다음 5가지 약점이 있는지 점검:"* → 5 시각 한꺼번에 검토
3. **Strategist 3단계 최종**: "BUY 72% (78%에서 하향) / 적정가 78,000원" + Risk 섹션에 자체 발견 약점 4가지 명시

### 3.3 장점 (Pros)

| # | 장점 | 설명 |
|---|------|------|
| ✅ | **LLM 호출 절감** | Strategist 1회 호출 안에 모든 검증 포함. 총 13회 유지 |
| ✅ | **비용 안전** | 36원/회 그대로. 5만원 상한 여유 있음 |
| ✅ | **응답 시간 ↑X** | 병렬 동기화 없음. 단일 호출 |
| ✅ | **구현 단순** | 시스템 프롬프트 1개에 "5 시각 자체 검토" 지시문 추가 |
| ✅ | **Phase 1 가능** | 8주차 MVP에 무리 없이 포함 |
| ✅ | **검증 결과 일관성 ↑** | 같은 LLM 인스턴스라 시각 일관성 유지 |

### 3.4 단점 (Cons)

| # | 단점 | 설명 |
|---|------|------|
| ❌ | **발표 임팩트 ↓** | "Critic Agent 5인 페르소나" 한 줄 자랑 못 함 |
| ❌ | **다양성 효과 ↓** | 같은 LLM이 자기 답을 검토 → 자기방어 편향 가능 |
| ❌ | **A2A 사례 부족** | 10주차 강사 요구 "A2A 적용" 의 사례를 다른 곳에서 찾아야 함 (예: Curator ↔ Strategist 종목 추천 협상) |
| ❌ | **검증 누락 위험** | "5 시각 검토" 가 프롬프트 1줄로 처리되면 실제 검토가 깊지 않을 수 있음 |
| ❌ | **모듈성 ↓** | Critic 끄기/켜기 = 프롬프트 수정 필요 (코드 수정 X 이지만 명확하지 않음) |

### 3.5 비용 추산 (통합안 기준)

| 항목 | 통합 안 |
|------|---------|
| 1회 호출 수 | 13회 (Strategist 1회 안에 검증 포함, output 토큰 ~30% 증가) |
| 1회 비용 | 36원 + α (Strategist output 1.3배 → +~5원) ≈ 41원 |
| 12주 1,000회 | 41,000원 |
| **5만원 상한 안전성** | ✅ 안전 |

### 3.6 구현 예시 (통합 안)

```python
# src/stock_agent/prompts/strategist/system_v1.md
"""
당신은 투자 분석가입니다. 다음 4 워커 결과를 종합해 BUY/HOLD/SELL을 결정합니다:

[1단계] 1차 결정을 내리세요.
[2단계] 다음 5 시각으로 1차 결정을 자체 검토하세요:
  1) 회계 보수주의자
  2) 매크로 비관론자

  각 시각에서 *반박 가능한 점* 1개씩 적으세요.
[3단계] 검토 결과를 반영해 최종 결정을 내리세요.
[4단계] WHAT·HOW MUCH·WHY·RISK 4슬롯으로 출력하세요.
```

## 5. PM 권고 (Hybrid 안)

> **둘 중 하나가 아니라 단계적 접근을 권고합니다.**

### Phase 1 (7~8주, MVP)
- **B 안 (통합)** 으로 시작
- Strategist 안에 5 시각 자체 검토 1단계 포함
- 비용 안전 + 일정 안전 + Phase 1 진입 빠름

### Phase 2 (9~11주, 고급기능)
- **A 안 (분리)** 으로 점진 전환
- 10주차 "A2A 적용" 강사 요구의 핵심 사례로 배치
- 발표 임팩트 강화 + 다양성 효과 확보

### Phase 2 진입 조건
- 비용 모니터링 결과 월 3만원 미만 안정 유지 시 → A 안 도입
- 월 3만원 초과 시 → B 안 유지하고 발표는 "단일 검증으로 충분한 결과 품질 확보" 메시지로
